지시문 — Import Preview 등록 전 안전장치 및 검토 UX 강화

현재 상품 마스터 Import Preview는 parser/canonical/product-option 분석 구조가 어느 정도 안정화된 상태입니다.

이번 작업의 목적은 기능 확장이 아니라, 운영자가 실수로 대량 등록하지 않도록 안전장치와 검토 UX를 강화하는 것입니다.

중요:
- DB schema 수정 금지
- parser logic 수정 금지
- canonical dictionary 수정 금지
- API 구조 대규모 변경 금지
- 이번 작업은 Preview 화면의 등록 UX/안전장치 중심으로만 수정

현재 문제:
신규 항목이 500건 이상 표시될 수 있어, 운영자가 실수로 전체 등록을 누르면 검토필요 항목까지 한 번에 INSERT될 위험이 있습니다.

수정 요청:

1. Import Preview 등록 버튼 구조 개선

현재 단일 등록 버튼이 있다면 다음 3개 방식으로 분리해 주세요.

- 선택 항목만 등록
- 검토필요 제외하고 등록
- 전체 등록

단, “전체 등록”은 위험한 버튼이므로 강조하지 말고 보조 버튼 스타일로 배치해 주세요.

2. Review Reason 있는 항목 보호

다음 조건 중 하나라도 해당하는 항목은 기본 자동등록 대상에서 제외해 주세요.

- reviewReason 존재
- confidence가 낮음
- 미지원 언어 포함
- Product 불명확
- 다중 산업 감지

“검토필요 제외하고 등록” 버튼은 위 항목들을 제외한 안전 항목만 등록 대상으로 계산해야 합니다.

3. 등록 전 확인 모달 추가

어떤 등록 버튼을 누르더라도 바로 등록하지 말고 확인 모달을 띄워 주세요.

모달에는 다음 정보를 표시해 주세요.

- 총 Preview 항목 수
- 실제 등록 대상 수
- 제외되는 검토필요 항목 수
- reviewReason 포함 항목이 등록 대상에 포함되는지 여부
- 등록 후 되돌리기 어렵다는 경고 문구

특히 “전체 등록”을 선택한 경우에는 다음 경고를 강하게 표시해 주세요.

“검토필요 항목까지 포함하여 전체 등록됩니다. 실제 운영 상품 마스터에 반영되므로 신중히 확인하세요.”

4. 선택 등록 기능

Preview table 각 행에 checkbox를 추가해 주세요.

- header checkbox로 현재 탭/현재 필터 결과 전체 선택 가능
- 개별 행 선택 가능
- 선택 항목 수 표시
- “선택 항목만 등록” 버튼은 선택된 항목이 없으면 disabled 처리

5. Confidence / Review 필터 추가

운영자가 검토 대상을 쉽게 찾을 수 있도록 Preview 상단에 최소한의 필터를 추가해 주세요.

필터:
- 전체 보기
- 검토필요만 보기
- 등록 가능만 보기
- Confidence 80 미만만 보기

정렬:
- 낮은 confidence순
- 높은 confidence순

기본 정렬은 낮은 confidence 또는 reviewReason 있는 항목이 위에 오도록 해 주세요.

6. 탭/카운트 표시 개선

상단 Preview 영역에 다음 카운트를 표시해 주세요.

- 신규
- 중복
- 검토필요
- 등록 가능
- 선택됨

카운트는 필터/선택 상태와 혼동되지 않도록 명확히 표시해 주세요.

7. 기존 Sticky Header 유지

이전 작업에서 수정한 Preview table sticky header 구조는 유지해 주세요.

중요:
- overflowY와 overflowX를 단일 wrapper에 유지
- sticky는 tr이 아니라 th에 직접 적용
- position: sticky; top: 0; zIndex 적용
- 이번 작업으로 sticky header가 다시 깨지지 않도록 확인

8. 검증

수정 후 다음을 확인해 주세요.

- Babel/TypeScript 빌드 통과
- Import Preview 화면 렌더링 정상
- 신규/중복/검토필요 탭 전환 정상
- sticky header 유지
- 선택 checkbox 동작 정상
- 선택 항목만 등록 버튼 disabled/enabled 정상
- 검토필요 제외 등록 시 reviewReason 있는 항목 제외
- 전체 등록 시 강한 경고 모달 표시
- 실제 등록 API 호출 전 등록 대상 수가 모달에 정확히 표시됨
