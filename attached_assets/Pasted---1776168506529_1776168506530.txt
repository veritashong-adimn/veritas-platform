[목표]
상품/단가 관리 기능을 “자유 입력 구조”에서 “자동 생성 + 통제 + 표준화 구조”로 개선한다.
데이터 일관성, 중복 방지, 향후 자동매칭/자동견적을 위한 기반을 만든다.

--------------------------------------------------

[1. 상품코드 체계 통일]

- 기존 상품코드 중 INT-001, TRN-001 등 비표준 코드 전부 수정
- 새로운 규칙으로 일괄 변환

규칙:
[서비스]-[언어]-[카테고리]-[번호]

서비스:
TR = 번역
IN = 통역

예:
TR-KOEN-GEN-001
IN-KOEN-SIM-001

- 기존 데이터 마이그레이션 스크립트 작성
- 코드 중복 발생하지 않도록 처리

--------------------------------------------------

[2. 상품코드 자동 생성 (입력 금지)]

- 상품코드 입력 필드 제거
- 서버에서 자동 생성

로직:
같은 서비스+언어+카테고리 기준
마지막 번호 조회 후 +1

예:
TR-KOEN-GEN-001 → 다음은 002

--------------------------------------------------

[3. 입력 방식 전면 변경 (자유 입력 금지)]

아래 필드는 전부 드롭다운/선택형으로 변경

- 서비스유형: TR / IN
- 언어쌍:
  KOEN (한→영)
  ENKO (영→한)
  KOCN (한→중)
  KOJA (한→일)

- 카테고리:
  번역:
    GEN (일반)
    TECH (기술)
    MED (의료)
    LAW (법률)

  통역:
    SIM (동시)
    CON (순차)
    MIT (미팅)
    EXH (전시)

- 단위:
  번역 선택 시 → 어절 / 글자 / 페이지 / 건
  통역 선택 시 → 시간

※ 직접 입력 불가

--------------------------------------------------

[4. 상품명 자동 생성]

- 상품명 자동 생성 로직 추가

형식:
[언어] + [카테고리]

예:
KOEN + GEN → 한영 일반번역
KOEN + SIM → 한영 동시통역

- 사용자는 수정 가능하지만 기본값 제공

--------------------------------------------------

[5. 중복 검사 기능 추가]

상품 생성/요청 시 아래 기준으로 중복 체크

조건:
service_type + language_pair + category + unit

이미 존재 시:
- "기존 상품이 존재합니다" 경고
- 기존 상품 보기 버튼 제공
- 그래도 진행 시 관리자 승인 필요

--------------------------------------------------

[6. 상품 등록 구조 변경]

- 기존 “상품 등록” → “상품 등록 요청”으로 변경

일반 사용자:
- product_requests 테이블에 저장

관리자:
- 승인 시 → products 테이블로 이동
- 코드 자동 생성

상태:
pending / approved / rejected

--------------------------------------------------

[7. 수정 제한 정책]

상품 수정 시 제한 적용

불가:
- code 수정
- service_type 변경
- language_pair 변경
- category 변경

가능:
- 상품명 수정
- 단가 수정

--------------------------------------------------

[8. 삭제 금지 → 비활성화]

- DELETE 금지
- is_active = false 처리

- 비활성화 시 사유 입력 필수

사유 예:
- 중복 상품 정리
- 사용 중단
- 코드 재정비

--------------------------------------------------

[9. 로그 시스템 추가]

아래 이벤트 로그 기록

- product_requested
- product_approved
- product_rejected
- product_created
- product_updated
- product_deactivated

--------------------------------------------------

[10. UI 개선]

- 상품 카드에서 서비스유형 표시 정확히 (번역/통역 혼동 제거)
- 잘못된 태그 표시 수정

- 검색 필터 추가:
  서비스유형
  언어쌍
  카테고리
  활성/비활성

--------------------------------------------------

[11. 엑셀 업로드 규칙 통일]

- 엑셀 업로드 시에도 동일 규칙 적용

필수:
- 코드 입력 금지
- 서비스/언어/카테고리 값 검증
- 중복 검사 수행

--------------------------------------------------

[완료 기준]

- 상품코드 자동 생성만 가능
- 코드 체계 100% 통일
- 중복 상품 생성 방지
- 상품 데이터 표준화 완료
- UI에서 번역/통역 구분 정확히 표시
