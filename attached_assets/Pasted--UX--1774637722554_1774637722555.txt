[목표]
프로젝트 목록 화면을 “최종 완성 UX” 수준으로 개선한다.
기능 변경 없이 시각적 구조, 밀도, 클릭 흐름을 최적화한다.

──────────────────────────────
1. 상태 / 재무 필터 “완전 분리 구조” 적용
──────────────────────────────

현재:
- 상태 / 재무가 나뉘어 있지만 하나의 영역처럼 보임

수정:

- 상태 필터 영역과 재무 필터 영역을 완전히 분리

구현 방식:

옵션 A (추천)
- 상태 필터 그룹
- divider (margin + border-top)
- 재무 필터 그룹

옵션 B
- 각각 카드(container)로 분리

추가:
- "상태", "재무" 라벨 font-weight 강화
- 재무 영역 background를 아주 약하게 다르게 처리 (예: bg-gray-50)

목표:
→ 두 필터는 완전히 다른 개념으로 인지되게 만들 것

──────────────────────────────
2. 상세 필터 “높이 축소 + 밀도 최적화”
──────────────────────────────

현재:
- 상세 필터가 과도하게 큼
- 입력 화면처럼 보임

수정:

- padding 축소 (p-5~6 → p-3~4)
- input height 축소 (h-10 → h-8)
- label 크기 축소 (text-sm → text-xs)
- 그룹 간 gap 축소

추가:

- 3열 구조 유지하되,
  카드 느낌보다 “플랫한 필터 바” 느낌으로 변경

목표:
→ “빠르게 쓰고 지나가는 필터 UI”로 만들 것

──────────────────────────────
3. 빠른 액션 “단일 CTA 구조”로 변경 (핵심)
──────────────────────────────

현재:
- 버튼 2~3개 혼재
- 사용자 판단 필요

수정:

각 row마다 대표 액션 1개만 강조

상태 → CTA:
- 접수됨 → [견적 생성]
- 견적 발송 → [견적 확인]
- 견적 승인 → [결제 등록]
- 결제 완료 → [번역사 배정]
- 진행 중 → [작업 보기]
- 완료 → [정산 확인]

추가 규칙:

- 대표 버튼: Primary 스타일
- 나머지 버튼:
  → text 버튼 또는 hover 시 노출
  → 또는 dropdown (… 메뉴)

목표:
→ “생각 없이 클릭 가능” 상태 만들기

──────────────────────────────
4. 테이블 row 클릭 UX 강화
──────────────────────────────

수정:

- row 전체 클릭 가능
- hover 시 background 강조 (bg-gray-50)

주의:

- 버튼 클릭 시:
  e.stopPropagation 적용

목표:
→ 클릭 영역 확대 + 탐색 속도 증가

──────────────────────────────
5. 필터 영역 sticky 처리
──────────────────────────────

수정:

- 필터 영역 전체 sticky 적용

조건:

- position: sticky
- top: 0
- z-index: 10 이상
- background 유지

목표:
→ 스크롤 중에도 필터 유지

──────────────────────────────
6. 컬럼 시각 정리 (선택)
──────────────────────────────

현재:
- 견적유형 / 결제 → 재무 성격

개선 옵션:

옵션 A (유지)
- 그대로 둠

옵션 B (추천)

- "결제" 컬럼을 "재무 상태"로 변경
- badge 스타일 통일

예:
- 미수금 / 선입금 / 차감 → 동일 스타일 계열

목표:
→ 정보 해석 속도 향상

──────────────────────────────
7. 전체 UI 톤 정리
──────────────────────────────

적용:

- 불필요한 border 제거
- box-shadow 최소화
- flat 디자인 유지

원칙:

- “관리 시스템” 느낌 유지
- “마케팅 페이지 느낌 제거”
