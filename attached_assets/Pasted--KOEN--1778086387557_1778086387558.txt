지시문

상품/단가 관리와 통번역사 상세 단가관리의 언어 구조를 완전히 통일하고, 언어쌍 방식(KOEN 등)을 제거하여 장기 확장 가능한 출발언어/도착언어 구조로 전면 수정한다.

현재 문제:

* 상품/단가와 통번역사 상세 단가관리의 언어 목록이 다름
* 언어쌍(KOEN) 사전 생성 구조가 비효율적
* 언어 추가 시 언어쌍 조합 폭증 문제 발생
* dropdown 자동 닫힘으로 UX 불편
* 검색형 구조가 아니라 언어 증가 시 운영 어려움

목표:

* 상품/단가
* 통번역사 상세 단가관리
* 견적
* 필터
* 향후 AI 자동매칭

모두 동일한 언어 마스터 기반으로 통일.

━━━━━━━━━━━━━━━━━━

1. 언어쌍(KOEN) 구조 제거
   ━━━━━━━━━━━━━━━━━━

기존:

* 한→영(KOEN)
* 영→한(ENKO)

사전 생성 방식 제거.

모든 구조를 아래 방식으로 통일:

* sourceLanguage (출발언어)
* targetLanguage (도착언어)

2. 상품/단가 관리 수정
━━━━━━━━━━━━━━━━━━

상품 등록 화면 수정:

기존:

* 언어쌍 선택

제거.

변경:

* 출발언어
* 도착언어

2개 searchable select 추가.

검색 필터도 동일 수정:

* 출발언어 전체
* 도착언어 전체

구조 유지.

* 출발언어

5. searchable select 구조로 변경

6. dropdown 자동 닫힘 문제 수정

현재:

  또는
* searchable popover 방식으로 변경하여 outside click 오작동 제거.

7. 코드 생성 방식 수정
━━━━━━━━━━━━━━━━━━

기존:

변경:

또는:

자동 생성 규칙 수정.

규칙:
[상품유형]-[출발언어]-[도착언어]-[카테고리]-[번호]
