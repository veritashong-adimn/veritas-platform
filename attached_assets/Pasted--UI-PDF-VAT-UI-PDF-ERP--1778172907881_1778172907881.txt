[상품/견적서 UI·PDF 간소화 및 VAT 구조 개선 수정 지시문]

작업 목적:
견적 입력 UI와 PDF 견적서가 현재 ERP 화면처럼 복잡하게 보이는 문제를 해결하고,
실제 B2B 고객 견적서 스타일처럼 간결하고 가독성 높게 개선한다.

───────────────────────

1. PDF 견적서 단위 세로 출력 수정
   ───────────────────────

현재 문제:
PDF 견적서에서 단위가 아래처럼 세로로 출력되고 있음.

예:
1
시
간

페
이
지

세
트

원인:
PDF table cell width 부족 + word-break/line-break 강제 적용 때문.

수정 사항:

* 단위 컬럼 width 확대
* white-space: nowrap 적용
* word-break: keep-all 적용
* 줄바꿈 금지

결과 예시:

* 1시간
* 페이지
* 세트
* 1일

형태로 가로 출력되도록 수정.

───────────────────────
2. PDF 견적서 유효기간 문구 수정
───────────────────────

현재:
"본 견적서는 발행일로부터 14일간 유효합니다."

변경:
"본 견적서는 발행일로부터 30일간 유효합니다."

추가:

* 시스템 기본 견적 유효기간도 30일로 통일 유지.
* 하드코딩된 14일 문구 전체 제거.

───────────────────────
3. PDF 내부 상태값 제거
───────────────────────

현재:
"견적상태: 대기"

문제:

* 고객용 문서에 내부 상태 노출
* 신뢰감 저하
* 실무상 불필요

수정:
PDF 견적서에서 아래 항목 제거:

* 견적상태

유지:

* 프로젝트명
* 담당자
* 견적일

───────────────────────
4. 견적 입력 UI VAT 구조 전면 간소화
───────────────────────

현재 문제:
각 견적 항목마다:

* 부가세10%
* 면세
* 영세율

을 개별 선택하고 있음.

실무 문제:

* 입력 복잡도 증가
* 대부분 견적은 전체 기준 VAT 적용
* 항목별 VAT 혼합 케이스 매우 드묾

───────────────────────
5. VAT 선택을 견적 전체 기준으로 변경
───────────────────────

상단 "회계/세무 옵션" 영역에
"과세 유형" 항목 추가:

옵션:

* 부가세 10%
* 면세
* 영세율

기본값:

* 부가세 10%

설계:

* 견적 전체에 단일 VAT 정책 적용
* 항목별 VAT 선택 제거

───────────────────────
6. 항목별 VAT 드롭다운 제거
───────────────────────

현재 항목 구조:
수량 / 단가 / 부가세선택 / 금액

변경:
수량 / 단가 / 금액

만 유지.

삭제 대상:

* 항목별 VAT select
* 항목별 세액 계산
* 항목별 VAT 상태 표시

───────────────────────
7. 총액 계산 구조 변경
───────────────────────

현재:
각 항목별 VAT 계산 후 합산

변경:

1. 공급가액 합계 계산
2. 견적 전체 VAT 계산
3. 최종 총액 계산

계산식:

[부가세10%]
VAT = 공급가액 합계 × 0.1

[면세]
VAT = 0

[영세율]
VAT = 0
(단, PDF에는 "영세율 적용" 표시 가능)

───────────────────────
8. 견적 하단 합계 UI 정리
───────────────────────

현재:
항목별 세금 계산 때문에 복잡함.

변경 후 구조:

공급가액
부가세(VAT)
총금액

3줄 구조로 단순화.

예:
공급가액: 4,350,000원
부가세(VAT 10%): 435,000원
총금액: 4,785,000원

───────────────────────
9. PDF 견적서 공급내역 테이블 간소화
───────────────────────

현재:
품목 / 단위 / 수량 / 단가 / 공급가액 / 세액 / 합계

변경:
품목 / 단위 / 수량 / 단가 / 공급가액

만 표시.

VAT/총액은 하단 총계 영역에서만 표시.

───────────────────────
10. 기존 데이터 호환성 유지
───────────────────────

기존 견적 데이터 중:
item.vatType 존재 시에도
렌더링 오류 없이 처리.

마이그레이션:

* 기존 item-level vatType은 nullable 유지 가능
* 신규 생성 견적은 quote-level vatType 사용

───────────────────────
11. UI/UX 원칙 적용
───────────────────────

목표:
"입력 가능한 정보"가 아니라
"실무에서 실제 필요한 정보만 보이게" 수정.

원칙:

* 고객용 문서는 단순하게
* 내부 회계 옵션은 최소 노출
* VAT는 견적 전체 기준
* ERP 느낌 제거
* 실제 기업 견적서 스타일 유지

───────────────────────
12. E2E 테스트 최소 범위만 실행
───────────────────────

중요:
이번 작업은 UI 구조 수정이므로
전체 프로젝트 풀 E2E 금지.

허용:

* 견적 생성
* VAT 계산
* PDF 출력
* 총액 계산

핵심 시나리오만 검증.

금지:

* unrelated 전체 회귀 테스트
* 장시간 Playwright 반복 실행
* 중복 브라우저 테스트 루프

먼저 구현 후:
1회 수동 확인
→ 핵심 시나리오만 테스트
→ 완료 보고.

크레딧 과소모 방지 필수.
