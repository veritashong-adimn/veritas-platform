상품 마스터 taxonomy 1단계 — UX polishing 작업 지시문

중요:
이번 작업은 “2차(import 구조 안정화)” 작업이 아니다.

현재 완료된:
- 상품 taxonomy 4종(TR/IN/EQ/EX)
- 상품유형별 UI 분리
- 장비/실비 구조 분리

를 유지한 상태에서,
운영 workflow에 맞게 상품 등록 UX를 다듬는 polishing 작업만 진행한다.

==================================================
절대 금지
==================================================

이번 작업에서는 아래 작업 금지:

- import 구조 변경
- 엑셀 parser 변경
- products schema 변경
- quote_items 구조 변경
- 프로젝트 구조 변경
- 자동견적 구조 변경
- 번역사 매칭 구조 변경
- AI 추천 구조 변경
- DB 마이그레이션 추가
- taxonomy 재설계

==================================================
1. 장비 상품 — 기본 수량단위 자동 선택
==================================================

현재 장비 상품 등록 화면에는:
- 개
- 세트
- 부스

선택 UI가 존재한다.

여기서:
장비 대분류(mainCategory)에 따라
기본 수량단위가 자동 선택되도록 개선한다.

예시:

FM 장비
→ 기본값:
세트

적외선 장비
→ 기본값:
세트

리시버
→ 기본값:
개

부스
→ 기본값:
개

--------------------------------------------------

중요:
운영자가 직접 변경은 가능해야 한다.
자동 선택은 “초기 추천값” 역할만 한다.

==================================================
2. 실비 상품 — 단위 단순화
==================================================

실비(EX) 상품은 대부분:
- 배송/퀵
- 출장비
- 식비
- 교통비

등 “건” 단위를 사용한다.

따라서:

실비 상품 선택 시:
기본 단위를 자동으로:
건
으로 설정한다.

--------------------------------------------------

단위 dropdown은 유지 가능하지만:

- 페이지
- 시간
- 세트

등 번역/통역/장비용 단위는
실비에서 보이지 않도록 한다.

즉:
실비 전용 단위 목록만 표시.

예:
- 건
- 박
- 회

정도만 유지 가능.

==================================================
3. 통역 상품 — 진행시간 preset UX 개선
==================================================

현재:
“기본 진행시간”
placeholder만 존재한다.

이를 운영 workflow 기준으로 개선한다.

--------------------------------------------------

통역 상품 선택 시:
preset 버튼 추가:

- 2시간
- 반일(4h)
- 종일(8h)

--------------------------------------------------

버튼 클릭 시:
기본 진행시간 input에 자동 입력.

예:
반일 클릭
→ input:
4h

--------------------------------------------------

직접입력은 유지한다.

==================================================
4. 번역 상품 — 단위 preset 정리
==================================================

번역 상품은 실제 운영상:
- 페이지
- 단어
- 어절
- 글자

를 자주 사용한다.

따라서:

번역 상품 선택 시
단위 dropdown 기본 목록을:

- 페이지
- 단어
- 어절
- 글자

중심으로 정리한다.

--------------------------------------------------

반대로:
- 세트
- 개
- 박
- 시간

등 장비/실비/통역용 단위는
번역에서 노출하지 않는다.

==================================================
5. UI/UX 방향 유지
==================================================

현재 매우 잘 분리된 상태:

- 번역 → 언어/페이지 중심
- 통역 → 방향/시간 중심
- 장비 → 수량/기간 중심
- 실비 → 간단 비용 중심

이 구조를 절대 깨지 않는다.

이번 작업은:
“운영자 입력 UX polishing”
만 수행한다.

==================================================
6. 최종 확인
==================================================

확인할 것:

1.
번역 선택 시:
- 페이지/단어/어절/글자만 노출되는지

2.
통역 선택 시:
- 2h/반일/종일 preset 버튼 동작하는지

3.
장비 선택 시:
- FM → 세트 자동선택
- 리시버 → 개 자동선택
되는지

4.
실비 선택 시:
- 기본 단위 건 자동선택
- 페이지/시간/세트 안보이는지

5.
기존 taxonomy(TR/IN/EQ/EX)가 깨지지 않는지

6.
상품 등록/수정/검색 workflow 정상 동작하는지

7.
Babel parse / Vite HMR 오류 없는지 확인.
