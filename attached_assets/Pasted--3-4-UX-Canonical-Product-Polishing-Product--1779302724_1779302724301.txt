상품 마스터 3-4단계 — 운영 UX + Canonical Product Polishing 작업

중요:
이번 단계는
구조 변경 단계가 아니다.

현재 Product + Option 철학은 유지한다.

이번 작업은:

- 운영 UX polishing
- canonical product 안정화
- preview 신뢰도 향상
- review workflow 개선

목적이다.

즉:
운영자가 Preview 결과를
실무에서 빠르게 검토하고
신뢰할 수 있게 만드는 단계이다.

==================================================
절대 금지
==================================================

이번 단계에서도:

- products DB rewrite 금지
- 자동 merge 금지
- 자동 삭제 금지
- schema 변경 금지
- quote 구조 변경 금지
- 자동견적 로직 변경 금지

즉:
Preview 분석/운영 UX 개선만 수행.

==================================================
1. Preview Header Sticky (틀고정)
==================================================

현재 문제:

Preview를 아래로 스크롤하면
컬럼 제목이 사라진다.

운영자가:

- Product
- 언어쌍
- 방향
- 난이도
- 산업
- 상태

컬럼 의미를 잃게 된다.

--------------------------------------------------

수정 방향:

Preview table thead:

- position: sticky
- top: 0
- z-index 적용
- background 유지
- border/shadow 유지

--------------------------------------------------

중요:

Preview 영역 내부에서만 sticky.

전체 페이지 sticky 금지.

--------------------------------------------------

추가:

스크롤 시:
- header background 유지
- column alignment 깨짐 금지

==================================================
2. Product 후보 badge 명칭 개선
==================================================

현재 문제:

“번역 옵션”
표현이 어색하다.

--------------------------------------------------

수정 방향:

다음으로 통일:

[옵션화 가능]

--------------------------------------------------

예:

번역 [옵션화 가능]

--------------------------------------------------

중요:

Product 후보는:
항상 canonical product만 표시.

==================================================
3. Canonical Product Dictionary 강화
==================================================

현재 parser는
문자열 기반 추론 비중이 높다.

--------------------------------------------------

canonical product dictionary 강화.

--------------------------------------------------

예시 normalize:

일반번역      → 번역
전문번역      → 번역
법률번역      → 번역
의료번역      → 번역
인한전문번역   → 번역
번역사출장알바 → 번역
영어동시통역   → 동시통역
일한순차통역   → 순차통역

--------------------------------------------------

중요:

Product 후보는:
반드시 제한된 canonical set 안으로 normalize.

--------------------------------------------------

허용 Product 예시:

- 번역
- 감수
- 원어민감수
- 공증번역
- 동시통역
- 순차통역
- 수행통역
- 화상통역
- 전화통역

--------------------------------------------------

금지:

- 법률번역
- 전문번역
- 인한전문번역
- ko→en번역

등.

9. Confidence Score 구조 추가 (매우 중요)
==================================================

이번 단계 핵심.

--------------------------------------------------

confidenceScore 필드 추가.

--------------------------------------------------

예:

95 = 매우 확실
75 = 보통
45 = 애매
20 = 검토필요

--------------------------------------------------

예시 기준:

+ canonical product match
+ language pair match
+ difficulty match
+ industry match

높을수록 score 상승.

--------------------------------------------------

미지원 언어,
unknown service,
product ambiguity 있으면 감점.

--------------------------------------------------

중요:

이번 단계에서는
UI에 크게 노출하지 않아도 된다.

backend 분석 구조에 먼저 포함.

--------------------------------------------------

향후:

- AI 자동등록

- AI 추천
- BI
- 검토큐

기반으로 사용 예정.

11. 최종 확인
==================================================

반드시 확인:

1.
Preview header sticky 정상 동작하는지.

2.
스크롤 시 컬럼 제목 유지되는지.

3.
Product 후보가 canonical 형태만 남는지.

4.
“번역 옵션” 제거되고
“옵션화 가능”으로 변경됐는지.

5.
검토필요 reason chip 정상 표시되는지.

6.
multi-industry 감지 표시되는지.

7.
confidenceScore 계산되는지.

8.
Preview row density가 compact 유지되는지.

9.
기존 import workflow 정상 유지되는지.

10.
기존 DB 데이터 변경 없는지.

11.
Babel parse / Vite HMR 오류 없는지 확인.
