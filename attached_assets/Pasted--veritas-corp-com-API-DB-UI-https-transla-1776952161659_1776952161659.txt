[작업명]
veritas-corp.com 데이터 미반영 원인 분석 (API/DB 연결 확인)

[목표]
현재 동일한 UI임에도 불구하고

- https://translate-hub.replit.app → 데이터 정상 표시
- https://veritas-corp.com → 데이터 없음 (초기 상태)

현상이 발생하고 있다.

이는 코드 문제가 아니라
“API 서버 또는 DATABASE 연결 차이”일 가능성이 매우 높다.

veritas-corp.com이 어떤 API와 DB를 보고 있는지 정확히 확인한다.

--------------------------------------------------
1. 프론트엔드 API 호출 확인
--------------------------------------------------

브라우저에서 아래 수행:

1) https://veritas-corp.com 접속
2) F12 → 개발자 도구 열기
3) Network 탭 클릭
4) 페이지 새로고침
5) /api/ 로 시작하는 요청 클릭

확인할 것:

- Request URL
- Response 데이터

---

동일 작업을 아래에서도 수행:

https://translate-hub.replit.app

---

[비교 포인트]

- Request URL이 동일한가?
- 도메인이 다른 서버를 가리키고 있지 않은가?
- Response 데이터가 다른가?

--------------------------------------------------
2. API 서버 주소 확인
--------------------------------------------------

프론트 코드에서 API base URL 확인

검색:

- BASE_URL
- API_URL
- fetch('/api/...') 또는 axios 설정

확인:

veritas-corp.com에서 호출되는 API가

1) 상대경로 (/api/...)인지
2) 절대경로 (https://xxx/api/...)인지

---

[문제 가능성]

- veritas-corp.com → 다른 서버 API 호출 중
- translate-hub → Replit API 호출

--------------------------------------------------
3. 백엔드 환경변수 확인
--------------------------------------------------

Replit (현재 정상 데이터 있는 쪽):

- DATABASE_URL 값 확인

---

veritas-corp.com이 연결된 서버에서:

- DATABASE_URL 확인

---

[비교]

두 값이 동일한지 확인

---

[결론 판단]

- 다르면 → DB가 다른 것 (현재 문제 원인)
- 같으면 → API 서버 자체가 다른 것

--------------------------------------------------
4. 서버 위치 확인
--------------------------------------------------

veritas-corp.com이 연결된 서버가 어디인지 확인

가능성:

- Replit
- Railway
- Vercel
- 별도 VPS

---

확인 방법:

- DNS (Cloudflare)에서 origin 확인
- 또는 API 응답 헤더 확인

--------------------------------------------------
5. DB 데이터 확인
--------------------------------------------------

veritas-corp.com 쪽 DB에 직접 접속하여 확인:

- projects 테이블
- companies 테이블
- contacts 테이블

---

[결과]

- 데이터 없음 → 다른 DB 사용 중
- 데이터 있음 → API 문제

--------------------------------------------------
6. 최종 원인 정리
--------------------------------------------------

아래 중 하나로 반드시 귀결됨:

1) veritas-corp.com → 다른 DATABASE 사용
2) veritas-corp.com → 다른 API 서버 사용
3) 프론트에서 API endpoint가 다르게 설정됨

--------------------------------------------------
7. 해결 방향 (중요)
--------------------------------------------------

목표:

veritas-corp.com → translate-hub.replit.app과 동일한

- API 서버
- DATABASE

를 사용하도록 통일

---

권장 방식:

- Replit 서버 하나로 통합
- DATABASE_URL 동일하게 설정
- API endpoint 단일화

--------------------------------------------------
8. 완료 기준
--------------------------------------------------

다음이 모두 만족되면 완료:

- veritas-corp.com에서 데이터 정상 조회됨
- translate-hub와 동일 데이터 표시됨
- API endpoint가 동일 서버를 가리킴
- DATABASE_URL이 동일하게 설정됨
