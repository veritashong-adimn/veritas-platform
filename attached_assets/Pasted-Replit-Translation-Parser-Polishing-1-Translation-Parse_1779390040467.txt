2. 국가명 기반 상품 자동 review_needed 처리
목표

국가명은 언어가 아니므로
자동 direction 생성 금지.

COUNTRY_KEYWORDS 추가
const COUNTRY_KEYWORDS = [
  "스위스",
  "벨기에",
  "유럽",
  "동남아",
  "중동",
  "아프리카",
];
parser 처리 규칙

상품명에 COUNTRY_KEYWORDS 포함 시:

reviewNeeded = true;
reviewReason = "COUNTRY_NOT_LANGUAGE";
riskLevel = "high";

자동 적용.

중요

이 경우:

❌ direction 자동 생성 금지
❌ displayName 자동 생성 금지

즉:

스위스-한글번역

은:

productCandidate = "번역"
direction = null
displayName = 원본 유지
reviewNeeded = true

로 처리.

3. 개념형 상품(Product taxonomy) 분리
목표

언어 기반 상품이 아닌
서비스/도메인 기반 상품 분리.

원어민감수 처리

아래 패턴 감지 시:

원어민감수
다국어원어민감수
원어민 감수

자동 처리:

productCandidate = "원어민감수";
type = "translation";
direction = null;
reviewNeeded = false;
할랄번역 처리
할랄번역

감지 시:

productCandidate = "번역";
domain = "halal";
reviewReason = "DOMAIN_BASED";

추가.

direction은 nullable 허용.

4. nullable direction 허용 안정화
목표

모든 상품이 language pair를 갖는다고 가정하지 않도록 수정.

수정 대상

direction/displayName 생성부.

규칙

다음 상품들은:

원어민감수
다국어원어민감수
할랄번역

direction 없어도 정상 처리.

즉:

direction = null

허용.
