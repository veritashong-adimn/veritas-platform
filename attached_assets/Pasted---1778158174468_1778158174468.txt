지시문

통역장비 상품 구조를 실제 실무 견적 방식에 맞게 개편한다.

현재 문제:

* 통역장비 단위가 “대/일/건”처럼 혼합되어 있음
* 장비 수량 개념과 사용기간 개념이 분리되지 않음
* 실제 통역장비 견적 방식과 맞지 않음
* “채널” 같은 기술 개념은 현재 운영 단계에서 불필요하게 복잡함

목표:
실제 통역장비 견적 방식 기준으로:

* 장비 수량
* 사용기간

을 분리하여 실무형 구조로 변경.

━━━━━━━━━━━━━━━━━━

1. 통역장비 단위 구조 변경
   ━━━━━━━━━━━━━━━━━━

현재:
단일 단위 선택 구조

예:

* 대
* 일
* 건

변경:
통역장비 전용 구조로 분리.

━━━━━━━━━━━━━━━━━━
2. 수량 단위 필드 추가
━━━━━━━━━━━━━━━━━━

통역장비 상품에:
“수량단위” 필드 추가.

사용 값:

* 개
* 세트
* 부스

현재 단계에서는 여기까지만 사용.

“채널”은 제외.

━━━━━━━━━━━━━━━━━━
3. 사용기간 필드 추가
━━━━━━━━━━━━━━━━━━

통역장비 상품에:
“사용기간” 필드 추가.

사용 값:

* 반일
* 1일
* 2일
* 3일
* 사용자 직접입력

━━━━━━━━━━━━━━━━━━
4. 견적 표시 방식 변경
━━━━━━━━━━━━━━━━━━

견적 예시:

* 리시버 250개 × 3일
* FM장비 1세트 × 2일
* 통역부스 2부스 × 1일

형태로 표시 가능하게 구조 준비.

━━━━━━━━━━━━━━━━━━
5. 기존 단위 구조 제거
━━━━━━━━━━━━━━━━━━

통역장비에서:

* 대
* 일
* 건

혼합 단위 제거.

“건”은 사용하지 않음.

━━━━━━━━━━━━━━━━━━
6. 번역/통역 상품 영향 금지
━━━━━━━━━━━━━━━━━━

이번 수정은:
통역장비 유형에만 적용.

번역/통역 상품은 기존 단위 유지:

예:

* 어절
* 페이지
* 1시간
* 반일
* 종일

정상 유지.

━━━━━━━━━━━━━━━━━━
7. 상품 등록/수정 UI 변경
━━━━━━━━━━━━━━━━━━

통역장비 선택 시만:

* 수량단위
* 사용기간

필드 표시.

다른 상품유형에서는 숨김.

━━━━━━━━━━━━━━━━━━
8. 상품 리스트 표시 개선
━━━━━━━━━━━━━━━━━━

통역장비 상품 카드에서:

예:

* 리시버 / 개 / 1일
* FM장비 / 세트 / 반일
* 통역부스 / 부스 / 3일

등으로 표시 가능하게 개선.

━━━━━━━━━━━━━━━━━━
9. 기존 데이터 migration
━━━━━━━━━━━━━━━━━━

기존:

* 대 → 개 또는 세트 기준으로 안전 migration
* 일 → 사용기간으로 migration
* 건 → 제거

기존 상품 깨지지 않게 처리.

━━━━━━━━━━━━━━━━━━
10. 불필요한 기술 개념 제외
━━━━━━━━━━━━━━━━━━

현재 단계에서는:

* 채널
* 다채널
* 기술채널수

등 복잡한 국제회의 기술 개념은 추가하지 않는다.

현재 운영 실무 기준:

* 장비 수량
* 부스 수량
* 사용기간

중심으로 단순 운영.

━━━━━━━━━━━━━━━━━━
11. E2E 최소 테스트 원칙 적용
━━━━━━━━━━━━━━━━━━

이번 작업은 중규모 UI 수정이므로 과도한 전체 E2E 반복 금지.

원칙:

* 관련 파일만 최소 탐색
* data-testid / aria-label 먼저 적용
* 빌드/HMR 우선 확인
* 핵심 시나리오만 테스트
* 동일 원인 2회 실패 시 즉시 중단 후 보고
* 전체 로그인→전체 플로우 반복 금지

━━━━━━━━━━━━━━━━━━
12. E2E 확인
━━━━━━━━━━━━━━━━━━

반드시 확인:

* 통역장비에서 수량단위 표시
* 사용기간 표시
* 번역/통역 상품 영향 없는지 확인
* 상품 등록 정상
* 상품 수정 정상
* 상품 리스트 정상 표시
* 기존 데이터 정상 migration
* 빌드 오류 없는지 확인

E2E 결과까지 보고.
