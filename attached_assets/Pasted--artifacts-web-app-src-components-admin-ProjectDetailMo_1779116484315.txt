artifacts/web-app/src/components/admin/ProjectDetailModal.tsx

관련:
견적 생성 / 수정 영역
회계/세무 옵션 accordion

목표:
현재 구조는:

* 결제/입금 방식
* 청구 시점

정도만 분리되어 있다.

하지만 실제 B2B 운영에서는:

* 입금
* 세금계산서 발행

이 서로 독립적으로 움직이며,
여러 단계로 나뉘는 경우가 많다.

즉:
현재 필요한 것은 단순:
“청구방식”

이 아니라,

# 실제 운영 workflow 기준의

# 결제/발행 스케줄 구조

이다.

중요:
이번 작업은:
회계 ERP를 만드는 것이 아니라,

# 실제 프로젝트 돈 흐름 운영

을 관리하기 위한 구조를 만드는 것이다.

---

1. 현재 문제

---

현재 구조는:

* 결제수단
* 입금방식
* 청구 timing

정도만 분리 가능.

하지만 실제 운영에서는:

* 입금은 여러 번 가능
* 세금계산서도 여러 번 가능
* 입금과 발행 timing이 서로 다를 수 있음

즉:
현재 구조만으로는
실제 운영 케이스를 충분히 표현하기 어려움.

---

2. 현재 실제 발생 케이스

---

예시:

① 단일 결제

* 100% 입금
* 세금계산서 1회 발행

② 선입금 + 잔금 + 각각 세금계산서

* 50% 선입금 → 세금계산서 발행
* 잔금 입금 → 세금계산서 발행

③ 선입금 + 잔금 후 일괄 발행

* 50% 선입금
* 잔금 입금
* 전체 세금계산서 1회 발행

④ 선금 / 중도금 / 잔금

* 계약금
* 중도금
* 잔금

각각:

* 금액 다름
* 날짜 다름
* 발행 timing 다름

---

3. 기존 구조 유지

---

현재 구조 유지:

① 문서구분
② 결제/입금 방식
③ 청구 시점

현재 방향 유지.

---

4. 신규 핵심 구조 추가 (중요)

세금계산서 발행 흐름을
관리하는 구조.
