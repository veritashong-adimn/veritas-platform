레플릿 지시문

이전 수정으로 backdrop 클릭 닫힘은 제거했지만,
번역 데이터 상세 패널이 Windows 캡처도구 실행/클릭 시 아직도 닫히는 문제는 해결되지 않았다.

즉, 이번에는 외부 클릭 문제가 아니라
blur / focusout / visibilitychange / selection reset / state reset 중 하나가 실제 원인일 가능성이 높다.

이번 작업은 추정으로 수정하지 말고,
실제 닫힘 원인을 로그로 확인한 뒤 정확히 제거하라.

1. 우선 실제 닫힘 원인을 찾기

번역 데이터 상세 패널이 닫히는 흐름을 전부 추적하라.

아래 항목을 검색하고, 번역 데이터 패널과 연결된 코드 기준으로 모두 확인하라.

setSelectedTranslationUnit(null)
setSelectedUnit(null)
setOpen(false)
onClose
onBlur
focusout
window.blur
document.visibilitychange
document.hidden
document.hasFocus
mouseup
mousedown
pointerdown
click
selection change
리스트 재조회 후 selected state 초기화
필터 변경/리렌더 시 selected state reset

중요:
이번에는 “아마 이거 같다”가 아니라
실제로 어떤 코드가 selected state를 null로 만드는지 확인하라.

2. 디버깅 로그 먼저 넣기

상세 패널이 닫히는 모든 경로에 로그를 넣어라.

예시:

console.log("[translation-panel] close reason: manual close button");
console.log("[translation-panel] close reason: ESC");
console.log("[translation-panel] close reason: blur");
console.log("[translation-panel] close reason: visibilitychange", document.hidden);
console.log("[translation-panel] close reason: selected reset from list rerender");
console.log("[translation-panel] selected unit changed:", selectedUnit?.id);

특히 아래 함수/이벤트에는 반드시 로그를 넣어라.

패널 닫기 버튼
ESC 처리
selected unit 초기화 시점
데이터 재조회 시점
패널 렌더링 조건 변경 시점
blur/focus/visibility 관련 이벤트
3. 가장 가능성 큰 원인 우선 점검

현재 증상상 가장 가능성 높은 것은 아래 둘 중 하나다.

가능성 A

캡처도구 실행 시 브라우저 포커스가 잠깐 벗어나고,
그때 blur / visibilitychange / focusout 계열에서 패널을 닫음

가능성 B

캡처도구 클릭 순간 리스트 영역이 다시 클릭 처리되거나 리렌더되면서
selectedTranslationUnit 상태가 null로 초기화됨

이 두 가지를 우선 확인하라.

4. 번역 데이터 상세 패널은 반드시 “수동 닫기 전용”으로 바꿔라

번역 데이터 상세 패널은 아래 경우에만 닫히게 하라.

X 버튼 클릭
명시적 닫기 버튼 클릭
ESC 키 입력

아래 경우에는 절대 닫히면 안 된다.

브라우저 포커스 이탈
캡처도구 실행
화면 다른 곳 클릭
리스트 리렌더
데이터 재조회
필터/검색 값 변경
창 전환
5. selected state를 리렌더와 분리하라

현재 번역 데이터 패널이 selectedTranslationUnit 같은 state에 의존할 텐데,
리스트 재조회나 부모 컴포넌트 리렌더 때문에 이 값이 초기화될 가능성이 있다.

아래를 점검하고 수정하라.

확인할 것
데이터 fetch 후 selected state를 무조건 null로 초기화하는 코드가 있는지
필터/검색 입력 시 selected state를 초기화하는 코드가 있는지
목록 새로고침 시 선택 상태가 사라지는지
상세 패널 렌더링 조건이 리스트 데이터 존재 여부와 강하게 묶여 있는지
수정 원칙
사용자가 직접 닫기 전까지 selected state 유지
데이터 재조회 후에도 선택한 id가 아직 존재하면 패널 유지
상세 패널은 목록 리렌더와 독립적으로 유지

예시 방향:

const [selectedTranslationUnitId, setSelectedTranslationUnitId] = useState<number | null>(null);
const selectedTranslationUnit = translationUnits.find(
  (u) => u.id === selectedTranslationUnitId
) ?? null;

그리고 데이터 재조회 시에도
선택한 id가 목록에 남아 있으면 패널을 닫지 마라.

6. blur / visibilitychange 로직 있으면 제거

번역 데이터 화면 또는 공통 훅/공통 모달 코드에 아래 같은 로직이 있으면 제거하라.

if (!document.hasFocus()) closePanel();
if (document.hidden) closePanel();
window.addEventListener("blur", closePanel);
document.addEventListener("visibilitychange", ...);

이런 방식은 캡처도구 사용 시 패널이 닫히는 대표 원인이다.

7. 패널 구현을 “고정 우측 패널”로 유지하라

번역 데이터 상세 패널은 모달처럼 자동 dismiss 되는 구조가 아니라,
선택한 항목의 상세를 보여주는 고정 우측 패널로 유지하라.

즉:

selectedTranslationUnitId가 있으면 렌더링
자동 close 없음
backdrop 없음
outside click dismiss 없음
blur dismiss 없음

필요하면 일반 fixed div 기반으로 단순화하라.

예시 방향:

{selectedTranslationUnitId && (
  <aside className="fixed right-0 top-0 h-full w-[520px] bg-white shadow-2xl z-50">
    ...
  </aside>
)}
