거래처 등록 화면 - 대표전화 / 휴대폰 / 이메일 / 웹사이트 필드 제거

[목표]
거래처 등록 화면에서
- 대표전화
- 휴대폰
- 이메일
- 웹사이트

항목을 제거하여,
거래처(회사) 기본 정보와 담당자 연락처 정보를 명확히 분리한다.

즉,
거래처 등록 화면은 회사 기본 식별 정보 중심으로 단순화하고,
연락 관련 정보는 담당자 등록/관리에서 처리하도록 정리한다.

--------------------------------------------------
1. 문제
--------------------------------------------------

현재 거래처 등록 화면에는 아래 항목이 포함되어 있다.

- 대표전화
- 휴대폰
- 이메일
- 웹사이트

하지만 실무상 이 항목들은
회사 대표 연락처라기보다
실제 커뮤니케이션 담당자 정보와 혼동되기 쉽다.

특히
- 휴대폰
- 이메일
은 담당자 정보로 관리하는 것이 더 적절하며,
거래처 화면에 함께 두면 정보가 중복되고 관리 기준이 흐려진다.

--------------------------------------------------
2. 수정 목표
--------------------------------------------------

거래처 등록/수정 화면에서는
아래 정보만 남겨 회사 기본 정보 중심으로 정리한다.

유지 대상 예시:
- 거래처명
- 사업자등록번호
- 대표자명
- 등록일
- 업태
- 종목
- 주소
- 메모

제거 대상:
- 대표전화
- 휴대폰
- 이메일
- 웹사이트

--------------------------------------------------
3. UI 수정
--------------------------------------------------

거래처 등록 화면에서 아래 입력 필드를 완전히 제거할 것.

제거 필드:
- 대표전화
- 휴대폰
- 이메일
- 웹사이트

즉, 화면에 보이지 않도록 삭제

주의:
단순 hidden 처리 말고,
폼 구조 자체에서 제거해 레이아웃도 정리할 것.

--------------------------------------------------
4. 레이아웃 정리
--------------------------------------------------

필드 제거 후 빈 공간이 생기지 않도록
입력 폼 레이아웃을 재정렬할 것.

권장 예시:
1행: 거래처명
2행: 사업자등록번호 / 대표자명 / 등록일
3행: 업태 / 종목
4행: 주소
5행: 메모

즉,
연락처 계열 필드 제거 후에도
화면이 어색하게 비지 않도록 간격과 그리드를 정리할 것.

--------------------------------------------------
5. 저장 로직 정리
--------------------------------------------------

거래처 등록/수정 시
아래 필드는 프론트 payload에서 제외할 것.

- phone
- mobile
- email
- website

백엔드가 해당 필드를 optional로 받더라도,
이번 작업에서는 거래처 폼에서 더 이상 보내지 않도록 정리

--------------------------------------------------
6. 기존 데이터 처리
--------------------------------------------------

중요:
이번 작업은 “입력 UI 제거”가 목표이며,
기존 DB 컬럼 삭제는 이번 범위에서 제외한다.

즉,
- DB 컬럼은 그대로 둘 수 있음
- 기존 데이터도 유지 가능
- 다만 신규 등록/수정 화면에서는 편집하지 않음

필요 시 추후 별도 작업으로
거래처 연락처 필드 정리/이관 여부를 검토

--------------------------------------------------
7. 수정 화면에도 동일 반영
--------------------------------------------------

거래처 신규 등록뿐 아니라
거래처 수정 화면에도 동일하게 반영할 것.

즉,
- 신규 등록 화면
- 거래처 상세 수정 화면

둘 다 아래 필드 제거:
- 대표전화
- 휴대폰
- 이메일
- 웹사이트
