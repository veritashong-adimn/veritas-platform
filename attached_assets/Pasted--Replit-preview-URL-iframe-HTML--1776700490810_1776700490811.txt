지시문

목표:
인사이트 관리 화면의 “자동 보완 실행” 버튼 클릭 시 뜨는 확인 팝업에
Replit preview URL, iframe 문구, HTML/페이지 내용처럼 보이는 이상한 문자열이 노출되는 버그를 수정한다.

이번 작업은 기능 추가가 아니라
👉 “팝업 메시지 오염 제거 + 사용자용 문구 고정” 작업이다.

문제 현상:
현재 자동 보완 실행 버튼 클릭 시 확인 팝업에 아래처럼 사용자에게 불필요한 문자열이 함께 보인다.

* replit.dev 주소
* “삽입된 페이지 내용” 같은 문구
* 페이지 전체가 섞인 듯한 이상한 문자열

이건 정상 동작이 아니고,
팝업 메시지에 문자열 대신 HTML/문서/응답 텍스트/iframe 관련 값이 잘못 들어간 버그로 판단된다.

수정 목표:
팝업에는 오직 아래 사용자 안내 문구만 깔끔하게 보여야 한다.

표시 문구:
현재 목록의 PARTIAL/NONE 인사이트(최대 20개)에 대해 자동 보완을 실행합니다.
시간이 걸릴 수 있습니다. 계속하시겠습니까?

---

1. 점검 대상

자동 보완 실행 버튼 클릭 시 호출되는 코드 전체를 확인한다.

특히 아래를 우선 점검:

* onClick 핸들러
* confirm(), alert(), modal open 코드
* message/content/description/title 변수
* fetch 응답을 text()로 읽어 그대로 팝업에 넣는 코드
* document.body.innerHTML, outerHTML, iframe 관련 문자열, location.href 등을 메시지로 합치는 코드

---

2. 수정 원칙

반드시 지킬 것:

* 확인 팝업 메시지는 하드코딩된 순수 문자열로 고정
* API 응답 본문을 팝업 메시지로 사용하지 말 것
* HTML, DOM, iframe, URL, location 정보 등을 팝업 메시지에 넣지 말 것
* window.confirm을 쓰든 커스텀 모달을 쓰든, 사용자에게는 깨끗한 안내문만 보여야 한다

---

3. 권장 구현 방식

현재 confirm 계열 코드가 있다면 아래 형태로 단순화한다.

예시 문구:
const confirmMessage =
"현재 목록의 PARTIAL/NONE 인사이트(최대 20개)에 대해 자동 보완을 실행합니다.\n시간이 걸릴 수 있습니다. 계속하시겠습니까?";

그리고:

* window.confirm(confirmMessage)
  또는
* 커스텀 ConfirmModal에 title / description을 명시적으로 분리해서 전달

권장 문구 구성:

* title: 자동 보완 실행
* description:
  현재 목록의 PARTIAL/NONE 인사이트(최대 20개)에 대해 자동 보완을 실행합니다.
  시간이 걸릴 수 있습니다. 계속하시겠습니까?

---

4. 커스텀 모달 사용 시 주의

커스텀 모달이라면 아래를 확인:

* description에 ReactNode/HTML 전체가 들어가고 있지 않은지
* iframe wrapper나 preview page 객체를 stringify해서 넣고 있지 않은지
* 에러 객체 전체를 그대로 렌더링하고 있지 않은지
* children으로 문서 내용이 주입되고 있지 않은지

필요하면 description 타입을 string으로 제한하라.

---

5. 버튼 동작은 유지

이번 수정은 팝업 문구만 정리하는 것이다.
아래 동작은 그대로 유지:

* 확인 클릭 → batch-auto-enhance 실행
* 취소 클릭 → 아무 작업 안 함
* 처리 완료 후 목록 갱신
* 기존 자동 보완 로직 자체는 변경 금지

---

6. 추가 점검

같은 패턴의 문제가 다른 곳에도 있으면 같이 점검:

* 단건 자동 보완 버튼
* 적용 / 무시 버튼 전 confirm 모달
* 삭제/병합 관련 확인 팝업

즉, “팝업 메시지에 응답/HTML이 그대로 들어가는 코드”가 있으면 이번에 함께 정리한다.

---

7. 완료 기준

수정 후 자동 보완 실행 버튼 클릭 시 팝업에는 아래만 보여야 한다:

* 제목: 자동 보완 실행
* 본문:
  현재 목록의 PARTIAL/NONE 인사이트(최대 20개)에 대해 자동 보완을 실행합니다.
  시간이 걸릴 수 있습니다. 계속하시겠습니까?

절대 보이면 안 되는 것:

* replit.dev URL
* 삽입된 페이지 내용
* HTML 태그
* 문서 전체 내용
* JSON stringify 결과
* 에러 객체 raw text
