지시문

목표:
현재 인사이트 관리 시스템에 “자동 보완 기능”을 추가하여,
AEO 점수가 낮은 인사이트를 사람이 수정하지 않아도 자동으로 보완되도록 한다.

이번 작업은 완전 자동화가 아니라,
👉 “자동 생성 + 반자동 승인 구조”를 만드는 1차 단계다.

핵심 개념:
문제 발견 (이미 완료)
→ 자동 보완 후보 생성
→ 운영자 검토
→ 적용

---

1. 자동 보완 대상 정의

아래 조건을 만족하는 인사이트를 자동 보완 대상으로 판단한다.

조건:

* aeoStatus = PARTIAL 또는 NONE
* faqCount < 3 OR relatedCount = 0

추가 조건:

* sourceType = data 또는 blog
* drop 상태는 제외

---

2. 자동 보완 필드

이번 단계에서 자동 생성할 항목:

* faqJson (3~5개)
* relatedIds (3개 이상 추천)
* aeoTitle (없을 경우)
* aeoDescription (없을 경우)

주의:

* 기존 값이 있으면 덮어쓰지 말고 “보완 후보”로 따로 생성

---

3. DB 구조 추가

insights 테이블에 아래 필드 추가:

* autoFaqJson (json, nullable)
* autoRelatedIds (json, nullable)
* autoAeoTitle (text, nullable)
* autoAeoDescription (text, nullable)
* autoGeneratedAt (timestamp)

또는 별도 테이블 사용 가능:

insight_auto_suggestions

* id
* insightId
* type (faq / related / meta)
* payload (json)
* status (pending / applied / rejected)
* createdAt

👉 권장: 별도 테이블 (확장성 때문)

---

4. 자동 생성 로직 (백엔드)

트리거:

* 인사이트 생성 후
* 또는 “자동 보완 실행” 버튼 클릭 시

동작:

Step 1:
AEO 상태 검사

Step 2:
부족 항목 판단

* faq 부족
* related 없음
* meta 없음

Step 3:
AI 호출

프롬프트:

* question 기반
* shortAnswer 기반
* 기존 faqJson 참고

요청 내용:

* 유사 질문 3~5개 (faq)
* 관련 인사이트 추천 (텍스트 기준 → 이후 매칭)
* SEO 최적 title/description

Step 4:
결과 저장 (auto_* 또는 suggestion 테이블)

---

5. relatedIds 생성 방식

AI가 직접 ID를 만들지 않도록 한다.

프로세스:

* AI → 관련 질문 텍스트 생성
* 서버 → 기존 인사이트와 유사도 매칭
* 상위 3개 ID 추출 → autoRelatedIds 저장

---

6. 관리자 UI 추가

위치:
인사이트 상세 패널

섹션:
“자동 보완 제안”

표시 항목:

* FAQ 제안

  * 기존 vs 제안 비교
* 관련 인사이트 제안
* AEO Title/Description 제안

버튼:

* “적용”
* “무시”

---

7. 적용 로직

“적용” 클릭 시:

* faqJson ← autoFaqJson
* relatedIds ← autoRelatedIds
* aeoTitle ← autoAeoTitle
* aeoDescription ← autoAeoDescription

적용 후:

* suggestion status = applied
* aeoScore 재계산

---

8. 자동 실행 옵션 (선택)

환경 설정 추가:

* autoEnhanceEnabled (true/false)

true일 경우:

* 생성 직후 자동 보완 실행

false일 경우:

* 수동 버튼 기반

---

9. UI 추가 버튼

인사이트 리스트 상단:

* “자동 보완 실행” 버튼

동작:

* 현재 필터된 리스트 기준으로
* 부족한 인사이트 batch 처리

---

10. 주의사항

* 기존 데이터 절대 덮어쓰기 금지
* 항상 “제안 → 승인 → 적용” 구조 유지
* AI 결과 품질 검증 없이 바로 반영 금지
* API timeout 대비 (비동기 처리 고려)

---

11. 기대 결과

작업 완료 후:

* FAQ 0 → 자동으로 3~5개 생성됨
* 관련 없음 → 자동 연결 후보 생성됨
* 점수 낮음 → 자동으로 상승 가능

운영자는:

👉 “수정”이 아니라 “승인”만 하면 됨
