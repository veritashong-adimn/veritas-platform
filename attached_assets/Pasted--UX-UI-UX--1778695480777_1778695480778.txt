지시문

[견적 입력 UX 구조 개편 — 사전 구조설계 및 구현 계획 수립]

중요:
이번 작업은 단순 UI 수정이 아니라
“견적 입력 시스템 UX 구조 개편” 작업이다.

바로 구현부터 진행하지 말고,
먼저 현재 구조 분석 → 최종 UX 구조 설계 → 단계별 구현 계획 수립을 우선 진행한다.

이번 작업의 핵심 목표는:

입력 화면을 최대한 간결하게 만든다.

한 화면에 더 많은 견적 항목이 보이도록 한다.

스크롤 길이를 대폭 줄인다.

실제 운영/실무형 ERP 견적 입력 구조로 개선한다.

UI는 간결하게,
DB 저장은 상세하게 유지한다.

즉:

“입력 UI”와 “데이터 저장 구조”를 분리한다.

───────────────────────

[핵심 원칙]

현재 문제:

견적 항목 하나가 카드처럼 너무 큰 높이를 차지함
장비가 많아지면 화면이 지나치게 길어짐
통역/번역/장비 모두 세부 옵션이 기본 노출됨
입력 피로도가 큼
사용자가 복잡하게 느낄 가능성 높음

목표:

“Compact Row 기반 견적 입력 구조”

즉:

1 row = 1 line 원칙

기본은 최소 입력만 보이고,
예외/세부 옵션만 펼침(accordion) 구조로 변경.

4️⃣ 장비 Row 구조

현재:
각 장비마다 카드처럼 세로 확장됨.

문제:
장비가 많아질수록 화면이 지나치게 길어짐.

목표:
엑셀형 Compact Table Row 구조.

예시:

FM장비 | 세트 | 1일 | 수량 1 | 600,000
리시버 | 개 | 1일 | 수량 100 | 2,000
송신기 | 개 | 1일 | 수량 2 | 50,000

기본 표시:

[▼ 상세]

눌렀을 때만:

행사장소
설치시간
철수시간
메모

등 표시.

중요:
기존 “장비 공통 설정” 구조는 제거 방향 우선 검토.

현재는:
공통 설정 자체가 오히려 화면을 차지하고 복잡도를 높이는 상태.

Compact Row 구조만으로 충분한지 우선 검토.

5️⃣ UI 철학

반드시 유지할 것:

“보이는 것은 간결,
저장 데이터는 상세”

즉:

UI:
엑셀형 / 운영형 / 빠른입력

DB:
현재처럼 상세 필드 유지 가능

예:
UI는:
한국어↔영어 | 동시통역 | 4시간

만 보여도,

DB에는:

langA
langB
direction
location
출장여부
메모
장비포함여부

등 상세 저장 가능.

8️⃣ 매우 중요

이번 작업은:
“보기 좋은 UI”가 목적이 아니라

“실제 운영자가 빠르게 견적 입력하는 UX”

가 목적이다.

즉:
감성형 카드 UI보다
실무형 Compact 입력 UX를 우선한다.
