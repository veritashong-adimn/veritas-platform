통번역사 단가 구조 수정 후 엑셀 대량 등록 기능 구현
1. 목표
통번역사 엑셀 대량 등록 전에 단가 구조를 먼저 확장한다.
현재 단가 입력 구조에 아래 항목을 반영하고, 이후 엑셀 업로드에서도 같은 구조로 등록될 수 있게 한다.

2. 단가 DB 구조 수정
translator_rates 또는 기존 통번역사 단가 테이블에 아래 필드를 포함한다.
serviceType          // 번역, 통역, 감수, 편집, DTPsubServiceType       // 세부유형sourceLanguage       // 출발언어targetLanguage       // 도착언어unit                 // 단가단위price                // 단가currency             // KRW, USD, EURvatIncluded          // VAT 포함 여부minPrice             // 최소금액, nullablebaseHours            // 기본시간, nullableovertimeRate         // 추가시간 단가, nullableisDefault            // 기본단가 여부isActive             // 활성/비활성memo                 // 메모
minPrice는 지금 당장 UI에서 필수 입력하지 않는다.
다만 DB/API/엑셀 구조에는 포함하고, 값이 없으면 null로 저장한다.

6. 엑셀 업로드 API
샘플 다운로드
GET /admin/translators/sample-excel
기능:


위 컬럼이 포함된 샘플 엑셀 생성


한글 컬럼명 유지


빈 값 허용 컬럼 구분



업로드 및 검증
POST /admin/translators/upload-excel
기능:


엑셀 업로드


통번역사 기본정보 파싱


단가정보 파싱


필수값 검증


이메일/휴대폰 중복 검사


업무유형/세부유형/단가단위 검증


오류/중복/정상 데이터 미리보기 반환



최종 등록
POST /admin/translators/bulk-create
기능:


미리보기에서 검증 통과한 데이터만 등록


중복 처리 옵션 적용


중복 처리 옵션:
mode: "insert" | "skip" | "update"


insert: 신규 등록


skip: 기존 통번역사는 건너뛰기


update: 기존 통번역사 정보 업데이트



7. 검증 기준
필수값:
이름휴대폰이메일
단가 필수값:
업무유형세부유형단가단위단가
단, 단가정보가 비어 있는 통번역사도 등록 가능하게 한다.
단가정보가 일부 입력된 경우에만 단가 필수값을 검증한다.

8. 중복 기준
중복 판단 우선순위:


이메일


휴대폰


이름 + 휴대폰


이름 + 계좌번호


중복 발견 시 바로 차단하지 말고, 미리보기에서 중복 가능으로 표시한다.

9. 저장 방식
저장은 트랜잭션으로 처리한다.


통번역사 기본정보 저장


정산정보 저장 또는 업데이트


단가정보가 있으면 translator_rates에 저장


실패 시 해당 row는 실패 처리


전체 실패가 아니라 row 단위 결과 리포트 반환



10. 업로드 결과 리포트
등록 완료 후 아래 결과를 반환한다.
{  total: number,  inserted: number,  updated: number,  skipped: number,  failed: number,  rateInserted: number,  rateFailed: number,  errors: []}

11. 오류 엑셀 다운로드
POST /admin/translators/export-errors
기능:


오류 row만 엑셀로 다운로드


오류 사유 컬럼 추가


직원이 수정 후 재업로드 가능하게 구성



12. 중요 요구사항


주민번호, 계좌번호는 민감정보이므로 평문 노출 최소화


상세 화면에서는 마스킹 표시


DB 저장 시 암호화 가능 구조로 분리


업로드 로그 저장


누가, 언제, 몇 건 업로드했는지 기록


1,000명 이상 업로드 가능하도록 배치 처리 고려


한글 깨짐 방지


기존 수기 등록 기능은 유지
