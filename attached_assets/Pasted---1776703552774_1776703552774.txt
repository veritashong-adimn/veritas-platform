지시문

목표:
인사이트 생성부터 게시까지의 전 과정을 완전히 자동화하여,
조건을 충족한 인사이트는 운영자 개입 없이 자동으로 게시되도록 한다.

이번 작업은
👉 “반자동 시스템 → 완전 자동 엔진”으로 전환하는 최종 단계다.

---

1. Auto Publish 개념

현재 흐름:
생성 → 보완 → 자동 적용 → publish_ready → (수동 게시)

목표 흐름:
생성 → 보완 → 자동 적용 → 조건 충족 → **자동 게시 (published)**

---

2. 설정값 추가

환경 설정 또는 config 테이블:

autoPublishEnabled: boolean (default: false)

권장:

* 초기에는 false
* 테스트 후 true 전환

---

3. Auto Publish 조건

아래 조건을 모두 만족하면 자동 게시:

필수 조건:

* status = publish_ready
* aeoScore ≥ 80
* faqCount ≥ 3
* relatedCount ≥ 2
* classification != drop
* shortAnswer 존재

추가 권장 조건:

* duplicationScore 낮음
* sourceType != manual (선택)

---

4. 자동 게시 로직

위치:
auto-enhance 완료 후 / autoApplyAndSetStatus 내부

Step 1:
auto 적용 후 상태 = publish_ready

Step 2:
조건 체크

IF 조건 만족 AND autoPublishEnabled = true:

→ status = published
→ publishedAt = now()
→ slug 없으면 생성
→ 공개 상태(public = true)

---

5. slug 생성

이미 없을 경우:

* question 기반 slug 생성
* 한글 허용
* 최대 60자 + id suffix

예:
동시통역-비용-어떻게-결정되나요-123

---

6. 로그 기록

logs 테이블:

* auto_published
* auto_publish_skipped (조건 미달)

필드:

* insightId
* reason (조건 미충족 사유)

---

7. UI 반영

published 상태:

* 파란색 배지
* “자동 게시됨” 표시

publish_ready 상태:

* “게시 대기”

자동 게시 시:

* 토스트 메시지:
  “자동 게시 완료”

---

8. 예외 처리

* 이미 published면 skip
* manual 수정된 인사이트는 auto skip 옵션 고려
* 동일 slug 충돌 방지
* 게시 실패 시 rollback

---

9. 안전장치 (중요)

옵션 추가:

autoPublishThreshold (기본 80)

→ 점수 기준 조정 가능

옵션:

autoPublishDryRun (true/false)

true:

* 실제 게시 안 함
* 로그만 기록
