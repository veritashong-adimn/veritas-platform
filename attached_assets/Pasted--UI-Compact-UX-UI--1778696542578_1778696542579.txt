지시문

[견적 입력 UI 최종 Compact UX 정리 작업]

작업 목적:
현재 견적 입력 화면은 이전보다 많이 간결해졌으나,
아직 일부 UI 요소가 “입력창처럼 보이는 문제”,
“버튼/타입 강조 과다”,
“상품선택+직접입력의 공간 점유” 문제가 남아 있다.

이번 작업은:

입력 속도 향상
시각적 밀도 개선
세로 길이 감소
ERP 느낌 감소
실제 운영형 견적 입력기 UX 강화

를 목표로 한다.

중요:
DB 구조/API/PDF 저장 구조는 유지한다.
이번 작업은 “UI Compact 정리” 중심이다.

───────────────────────

상세 펼침 영역(▼) Compact 버튼화
───────────────────────

현재 문제:

각 row 마지막에 있는:

보라색 긴 ▼ 영역
파란색 긴 ▼ 영역

이 입력창처럼 보임.

현재:
[ ▼ ]

변경:

작은 compact 버튼 형태로 변경:

예시:
[상세 ▼]

또는:
[▼]

권장:
텍스트 포함 compact pill 버튼.

디자인 조건:

width auto
px-2 ~ px-3 수준
inline-flex
justify-center
height 기존 input과 동일
입력창 border 스타일 사용 금지
button 스타일로 명확히 구분

적용 대상:

번역 row
통역 row
장비 row
실비 row

───────────────────────

상품선택 + 직접입력 구조 Compact화
───────────────────────

현재 문제:

상품선택 dropdown과
“항목명 직접 입력” input이 항상 동시에 노출됨.

세로 공간 및 시각적 복잡도 증가.

변경 구조:

기본:
[상품 선택 ▼]

직접입력 버튼:
[+ 직접입력]

클릭 시만:
[항목명 직접 입력]

펼쳐짐.

동작 규칙:

상품 선택 시:

직접입력 input 숨김
직접입력값 clear 가능

“+ 직접입력” 클릭 시:

직접입력 input 표시
상품 dropdown 선택 해제 가능

직접입력 취소 가능:
[x 직접입력 닫기]

───────────────────────

타입 버튼(번역/통역/장비/실비) 시각 강도 축소
───────────────────────

현재 문제:

각 row마다:

border 강조
색 강조
반복 노출

로 인해 시선 분산 발생.

변경:

선택된 타입만 강조.

미선택 버튼:

연회색 배경
border 약화
text-gray-500 수준

선택된 버튼:

현재 컬러 유지 가능
다만 border 두께 과도하지 않게 조정

목표:
“현재 선택된 타입만 눈에 띄게”

───────────────────────

통역 row width 최적화
───────────────────────

현재:
언어/방향/방식 영역이 약간 넓음.

아래 항목 width 축소:

언어A
언어B
방향
통역방식
단위

목표:
한 줄 밀도 향상.

주의:
모바일 깨짐 없이 유지.

───────────────────────

상세영역(showDetail) 애니메이션 최소화
───────────────────────

현재 펼침 시 약간 어색한 움직임이 있을 수 있음.

변경:

transition duration 짧게
opacity 최소
height auto 자연스럽게

목표:
빠른 입력 중심 UX.

───────────────────────

Row 내부 정렬 통일
───────────────────────

모든 row 공통 정렬 규칙 적용:

세로 중앙 정렬
input height 통일
버튼 height 통일
숫자 input width 일관성 유지
공급가액 box 정렬 통일

특히:
장비 row와 번역 row의 공급가액 영역 width 맞추기.
