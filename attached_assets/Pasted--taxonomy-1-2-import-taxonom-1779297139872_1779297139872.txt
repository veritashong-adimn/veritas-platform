상품 마스터 taxonomy 1단계 — 통역/장비 부대요금 재분류 작업

중요:
이번 작업은 “2차 import 구조 안정화” 작업이 아니다.

현재 완료된 taxonomy 구조(TR/IN/EQ/EX)는 유지하면서,
실제 통번역 운영 workflow에 맞게
부대요금의 귀속 taxonomy를 수정하는 작업이다.

==================================================
핵심 방향
==================================================

기존에는:
출장비/보상비/취소비용 등을
실비(EX)로 분류하는 방향이 있었다.

하지만 실제 운영에서는:

- 통역사 출장비
- 이동일 보상비
- 통역 저작권료
- 취소보상비
- 대기료
- 연장료

등은
“통역 서비스 revenue의 일부”

로 운영된다.

마찬가지로:

- 장비 엔지니어 출장비
- 장비 설치비
- 장비 철수비
- 장비 운영비

등은
“장비 서비스 revenue의 일부”

로 운영된다.

따라서:
단순 EX(실비) taxonomy가 아니라,
각 서비스 유형(IN/EQ)에 귀속시키는 방향으로 수정한다.

==================================================
1. 통역(IN) 부대요금 추가
==================================================

다음 항목은 EX가 아니라
IN(통역) taxonomy로 이동한다.

예시:
- 통역사 출장비
- 이동일 보상비
- 통역 저작권료
- 취소보상비
- 대기료
- 연장료
- 야간 할증
- 휴일 할증

--------------------------------------------------

중요:
이 항목들은:
“일반 경비”
가 아니라

“통역 서비스 수행 조건에 따른 요금”

으로 취급한다.

==================================================
2. 장비(EQ) 부대요금 추가
==================================================

다음 항목은 EX가 아니라
EQ(장비) taxonomy로 이동한다.

예시:
- 장비 엔지니어 출장비
- 장비 설치비
- 장비 철수비
- 장비 셋업비
- 장비 운영비
- 장비 연장 사용료

--------------------------------------------------

중요:
이 항목들은:
“일반 경비”
가 아니라

“장비 운영 서비스 요금”

으로 취급한다.

==================================================
3. EX(실비) taxonomy 재정리
==================================================

EX에는:
순수 일반 경비성 항목만 유지한다.

예:
- 배송/퀵
- 택배
- 우편
- 식비
- 숙박비
- 교통비(일반)
- 인쇄/출력
- 기타 일반 경비

--------------------------------------------------

즉:
서비스 수행 종속 비용은
EX에 두지 않는다.

==================================================
4. 상품코드 방향 유지
==================================================

통역 부대요금:
IN prefix 유지

예:
- IN-TRIP-001
- IN-CANCEL-001
- IN-OT-001

--------------------------------------------------

장비 부대요금:
EQ prefix 유지

예:
- EQ-SETUP-001
- EQ-ENG-001
- EQ-EXT-001

--------------------------------------------------

EX는:
일반 실비만 유지.

==================================================
5. 중요한 운영 방향
==================================================

이번 작업의 핵심은:

“어떤 revenue/workflow에 귀속되는가”

기준으로 taxonomy를 정리하는 것이다.

즉:

통역 수행 종속
→ IN

장비 운영 종속
→ EQ

일반 행정/경비
→ EX

방향 유지.

==================================================
6. UI/UX 방향
==================================================

현재 상품관리 UI 구조:
- 번역
- 통역
- 장비
- 실비

4분류는 유지한다.

과도한 grid/table 구조 추가 금지.
ERP식 복잡 분류 금지.

==================================================
7. 기존 데이터 처리
==================================================

기존 EX에 있는:
- 출장비
- 취소보상비
- 엔지니어 출장비

등이 있으면:

의미 기준으로:
IN 또는 EQ로 재분류한다.

--------------------------------------------------

중요:
기존 상품 데이터 삭제 금지.
가능하면:
- taxonomy 변경
- 상품코드 변경
- category 이동

수준으로 처리.

==================================================
절대 금지
==================================================

- import parser 수정
- quote item 구조 변경
- 프로젝트 구조 변경
- 자동견적 구조 변경
- 번역사 매칭 변경
- AI 추천 변경
- DB 대규모 재설계
- workflow 변경

==================================================
최종 확인
==================================================

1.
통역 부대요금이 IN taxonomy로 이동했는지 확인.

2.
장비 운영 부대요금이 EQ taxonomy로 이동했는지 확인.

3.
EX에는 순수 일반 실비만 남았는지 확인.

4.
상품코드 prefix(IN/EQ/EX)가 taxonomy와 일치하는지 확인.

5.
상품관리 UI 정상 동작 확인.

6.
기존 상품 데이터 손실 없는지 확인.

7.
Babel parse / Vite HMR 오류 없는지 확인.
