[작업명]
견적 항목 구조 재설계 (번역/통역 분리 + 선입금 관리 + 금액 UX 개선)

---

[목표]
현재 견적 생성 화면은 기본 입력은 가능하지만,
실제 통번역 실무에서 필요한 정보(파일, 일정, 장소, 시간, 잔액 관리 등)가 부족하다.

이를 보완하여
- 선입금 관리 (입금/차감/잔액)
- 번역 항목 (파일 기반)
- 통역 항목 (일정 기반)
- 금액 입력 UX

까지 모두 반영된 “실제 운영 가능한 견적 시스템”으로 개선한다.

---

━━━━━━━━━━━━━━━━━━
1️⃣ 선입금 관리 UI 보강
━━━━━━━━━━━━━━━━━━

[문제]
현재:
- 잔액 표시만 있음
- 입금/추가 입금/이력 관리 UI 없음

---

[구현]

선입금 영역에 아래 버튼 추가:

[입금 등록]  [이력 보기]

---

[입금 등록 모달]

입력:
- 금액 (필수)
- 입금일 (기본 오늘)
- 메모 (선택)

동작:
- prepaid_transactions에 deposit 추가
- prepaid_balances 갱신

---

[이력 보기]

표 형태:

날짜 | 유형 | 금액 | 전 잔액 | 후 잔액 | 프로젝트 | 메모

---

[중요 규칙]

- 기존 거래 수정 ❌
- 수정은 adjust 거래로 처리

---

━━━━━━━━━━━━━━━━━━
2️⃣ 금액 입력 UX 개선
━━━━━━━━━━━━━━━━━━

[문제]
- 35000 → 컴마 없음
- 세금 선택 방식 어색

---

[구현]

1. 금액 입력

입력 중:
- 숫자만 입력

입력 완료(blur):
- 자동 포맷 적용

예:
35000 → 35,000

---

2. 세금 선택 UI 변경

기존:
- 면세 / 10%

변경:

- 부가세 포함 (10%)
- 면세
- 영세율

---

[기본값]

- 부가세 포함 (10%)

---

━━━━━━━━━━━━━━━━━━
3️⃣ 번역 항목 구조 개선
━━━━━━━━━━━━━━━━━━

[문제]
- 번역 파일 없음
- 작업 근거 부족

---

[구현]

번역 항목에 아래 추가:

[파일 첨부 영역]

- 파일 업로드 (다중 선택 가능)
- 파일 리스트 표시

예:
- contract_v1.docx
- spec.pdf

---

[저장 구조]

quote_item_files:
- id
- quoteItemId
- fileUrl
- fileName
- createdAt

---

[UI]

파일 영역:

[파일 선택] + 추가

4️⃣ 통역 항목 구조 분리 (핵심)
━━━━━━━━━━━━━━━━━━

[문제]
현재:
- 번역/통역 구분 없음
- 일정 정보 없음

---

[구현]

상품 선택 시 유형 분기:

- 번역
- 통역

---

[통역 선택 시 추가 필드 표시]

필수 입력:

- 통역 날짜
- 진행 시간 (예: 4h / 8h / 추가시간)
- 장소
- 언어
- 통역 유형 (동시 / 순차 / 미팅)

선택 입력:

- 출장 여부
- 장비 여부
- 메모

---

[UI 예시]

통역 항목:

날짜: [2026-05-08]
시간: [4h]
장소: [서울 COEX]
언어: [EN-KO]
유형: [동시]
