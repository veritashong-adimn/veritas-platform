3. 지역명 기반 ambiguity 처리
목표

홍콩어 / 대만어 같은 표현은
자동 direction 생성 금지.

REGION_LANGUAGE_KEYWORDS 추가
const REGION_LANGUAGE_KEYWORDS = [
  "홍콩어",
  "대만어",
];
처리 규칙

상품명에 REGION_LANGUAGE_KEYWORDS 포함 시:

reviewNeeded = true;
reviewReasons.push("REGION_LANGUAGE_AMBIGUOUS");
riskLevel = "high";

자동 적용.

중요

이 경우:

❌ direction 자동 생성 금지
❌ displayName 자동 생성 금지

즉:

홍콩어-한글번역

은:

direction = null
displayName = 원본 유지
reviewNeeded = true

처리.

4. MISSING_DIRECTION 감소
목표

정상 alias가 있는 경우
displayName/direction 자동 생성되도록 수정.

검증 대상

아래 입력이:

입력	기대 결과
한-터키어번역	한국어-터키어 번역
한-그리스어번역	한국어-그리스어 번역
라오스-한글번역	라오어-한국어 번역

으로 자동 생성되어야 한다.

처리 조건

srcCode + tgtCode 둘 다 존재 시:

direction = `${srcCode}→${tgtCode}`;
displayName = `${ISO_LABEL[srcCode]}-${ISO_LABEL[tgtCode]} 번역`;

정상 생성.
