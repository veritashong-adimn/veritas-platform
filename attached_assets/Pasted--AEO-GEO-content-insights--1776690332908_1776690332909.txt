[목표]
기존 블로그 글 본문을 입력하면,
AEO/GEO용 인사이트 초안을 자동 생성하여 content_insights에 저장할 수 있는
"블로그 글 → 인사이트 변환" 기능을 추가한다.

이번 작업 범위:

* 인사이트 관리 화면에서 블로그 글 입력
* 블로그 글 기반 인사이트 초안 자동 생성
* 생성 결과는 draft + internal_summary 로만 저장
* 자동 게시 금지
* 생성 후 기존 인사이트 관리 흐름(수정/승인/게시)으로 이어지게 한다

---

[1단계: 버튼 추가]

위치:

* 인사이트 관리 화면 상단 우측
* 기존 "인사이트 생성 ▾" 버튼 드롭다운 메뉴 안에 신규 항목 추가

메뉴 항목:

* ✍️ 수동 입력 생성
* ⚡ 데이터 기반 자동 생성
* 📝 블로그 글 변환   ← 신규

클릭 시:
→ "블로그 글 → 인사이트 변환" 모달 오픈

---

[2단계: 블로그 글 변환 모달 UI]

모달 제목:

* 블로그 글 → 인사이트 변환

입력 항목:

1. 블로그 제목 (선택)

* text input
* placeholder: 예) 동시통역 비용은 어떻게 결정될까?

2. 블로그 본문 (필수)

* large textarea
* 줄바꿈 유지
* placeholder: 블로그 글 본문 전체를 붙여넣으세요

3. 출처 URL (선택)

* text input
* placeholder: https://...

4. 생성 개수 (필수)

* select
* 옵션: 1개 / 3개 / 5개
* 기본값: 3개

안내 문구:

* 생성된 인사이트는 모두 초안(draft) 상태로 저장됩니다.
* 자동 게시되지 않으며, 인사이트 관리에서 검수 후 게시할 수 있습니다.
* 실제 운영 데이터가 없는 경우 가격 정보는 임의 생성하지 않습니다.

버튼:

* [초안 생성]
* [취소]

검증:

* 본문이 비어 있으면 생성 버튼 비활성화
* 본문이 너무 짧으면 생성 차단 (예: 300자 미만)
  → "본문이 너무 짧아 인사이트 생성이 어렵습니다." 표시

[3단계: API 추가]

신규 API:
POST /api/admin/content-insights/from-blog

request body 예시:
{
"title": "동시통역 비용은 어떻게 결정될까?",
"content": "...블로그 본문 전체...",
"sourceUrl": "https://...",
"count": 3
}

response 예시:
{
"created": 3,
"items": [
{
"id": 31,
"question": "동시통역 비용은 얼마인가요?",
"shortAnswer": "...",
"longAnswer": "...",
"serviceType": "interpretation",
"questionType": "price"
}
]
}

---

[4단계: DB 필드 확장]

content_insights에 아래 필드가 없으면 추가:

* sourceType (text)
* sourceTitle (text nullable)
* sourceUrl (text nullable)

블로그 기반 생성 시 저장값:

* sourceType = 'blog'
* sourceTitle = 입력한 블로그 제목
* sourceUrl = 입력한 URL

레거시 데이터는 null 허용

---

[5단계: 변환 로직]

블로그 본문을 분석해서 아래 작업 수행:

1. 핵심 주제 추론

* serviceType 추론:

  * translation / interpretation / equipment
* domain 추론:

  * legal / finance / medical / general 등
* languagePair 추론:

  * ko-en / en-ko 등 본문에 있으면 추출
* industry 추론
* useCase 추론

2. 질문 생성

* count 값에 맞춰 1 / 3 / 5개 생성
* questionType 후보:

  * price
  * definition
  * comparison
  * process
  * faq

예시:

* 동시통역 비용은 얼마인가요?
* 동시통역 비용에 영향을 주는 요소는 무엇인가요?
* 통역 장비는 언제 필요한가요?
* 번역 단가에 영향을 주는 요소는 무엇인가요?

3. shortAnswer 생성

* 1~2문장
* 결론형 문장
* 과장 금지
* 블로그 본문 내용에 근거할 것

4. longAnswer 생성

* 블로그 본문 요약/재구성
* 마크다운 허용
* 소제목/불릿 사용 가능
* 지나치게 장황하지 않게 정리

---

[6단계: 중요한 저장 원칙]

매우 중요:

블로그 기반 생성은 "설명형 인사이트"로 취급한다.

규칙:

* 블로그에 실제 숫자 근거가 명확하지 않으면
  avgPrice / minPrice / maxPrice를 자동 생성하지 않는다
* 추정 수치를 만들지 않는다
* sourceCount는 null 허용
* confidenceScore는 기본 0.4 또는 0.5 수준 사용
* status = 'draft'
* visibilityLevel = 'internal_summary'

즉:

* 운영 데이터 기반 인사이트와
* 블로그 기반 인사이트를 명확히 구분한다

---

[7단계: 프론트 생성 후 동작]

초안 생성 성공 시:

1. 모달 닫기
2. 인사이트 목록 자동 갱신
3. 생성된 첫 번째 인사이트 자동 선택
4. 우측 상세 패널 자동 오픈
5. 2초 정도 하이라이트
6. 토스트 표시:

* "블로그 글 기반 인사이트 초안 3건이 생성되었습니다."

---

[8단계: 인사이트 관리 화면 표시 보완]

목록 또는 상세 패널에서 아래 출처 정보 표시:

* 출처 유형(sourceType)
* 출처 제목(sourceTitle)
* 출처 URL(sourceUrl)

표시 예:

* 출처 유형: blog
* 출처 제목: 동시통역 비용은 어떻게 결정될까?
* 출처 URL: https://...

레거시 데이터는 값 없으면 "-" 처리

---

[9단계: 안전장치]

* 자동 게시 금지
* published 상태로 바로 저장 금지
* 블로그 입력 실패 시:

  * 토스트 표시
  * 콘솔 로그 기록
* 긴 본문도 처리 가능하도록 서버 타임아웃/에러 처리 주의
* 줄바꿈, 따옴표, 특수문자 저장 시 escape 안정성 확인

---

[10단계: 최종 결과]

인사이트 관리 화면에서:

1. 블로그 글 본문 붙여넣기
2. 질문/답변 초안 자동 생성
3. draft 상태로 저장
4. 기존 수정/승인/게시 흐름으로 운영

할 수 있는 구조를 만든다.

즉,
기존 블로그 콘텐츠를
AEO/GEO용 인사이트 자산으로 재활용하는 기능 완성
