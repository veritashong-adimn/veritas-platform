상품 마스터 2단계 — Import 구조 안정화 작업

중요:
이번 작업은
“상품 import 구조 안정화”
단계이다.

이미 완료된:
- taxonomy(TR/IN/EQ/EX)
- 상품유형별 UI 분리
- 실비/통역/장비 부대요금 taxonomy
- 커스텀 상품 요청 구조

를 유지한 상태에서,
실제 CRM 상품 데이터를 안정적으로 플랫폼에 대량 등록할 수 있도록
import workflow를 안정화한다.

==================================================
이번 단계 핵심 목표
==================================================

현재 목표는:

“기존 CRM 상품 데이터를
플랫폼 상품 마스터 DB에 안전하게 import 가능한 상태로 만드는 것”

이다.

즉:
- import validation
- 중복 방지
- preview
- dry-run
- 검토 필요 항목 분리

를 중심으로 진행한다.

==================================================
절대 금지
==================================================

이번 단계에서는 아래 작업 금지:

- taxonomy 재설계
- DB schema 대규모 변경
- quote item 구조 변경
- 프로젝트 구조 변경
- 자동견적 구조 변경
- 번역사 매칭 구조 변경
- AI 추천 구조 변경
- ERP식 대량 grid 구조 추가

==================================================
1. Import preview 구조 추가
==================================================

엑셀 업로드 시:
바로 DB insert 하지 않는다.

반드시:
preview 단계 먼저 표시.

Preview 항목 예시:
- 상품코드
- 상품명
- 유형(TR/IN/EQ/EX)
- 대분류
- 단위
- 단가
- 상태
- 검토 필요 여부

--------------------------------------------------

상태 예시:
- 신규 등록
- 기존 상품과 중복 가능
- taxonomy 검토 필요
- 코드 충돌
- 상품명 유사 중복

==================================================
2. Dry-run 모드 추가
==================================================

실제 insert 전에:
“시뮬레이션 검증”

가능하게 한다.

Dry-run 결과:
- 신규 몇 건
- 중복 몇 건
- 충돌 몇 건
- 검토 필요 몇 건

요약 표시.

--------------------------------------------------

중요:
Dry-run에서는 DB INSERT 금지.

==================================================
3. 중복 검출 강화
==================================================

다음 기준으로 중복 후보 탐지:

- 상품코드 동일
- 상품명 동일
- 상품명 normalize 동일
- 유형+대분류+단위 동일

--------------------------------------------------

예:
- FM장비
- FM 장비

같은 normalize 중복 탐지 가능하게.

--------------------------------------------------

중복은:
자동 삭제하지 않는다.

반드시:
- 중복 후보
- 검토 필요

로 표시.

==================================================
4. 검토 필요 항목 분리
==================================================

다음 항목은:
자동 등록하지 않고
“검토 필요” 상태로 분리 가능하게 한다.

예:
- taxonomy 불명확
- 상품명 너무 짧음
- 단위 없음
- 유형 판별 실패
- 유사 중복 과다
- 코드 충돌

--------------------------------------------------

검토 필요 탭 또는 필터 제공 가능.

==================================================
5. 상품명 normalize 구조
==================================================

중복 검출 정확도를 위해 normalize 적용.

예:
- 공백 제거
- 대소문자 normalize
- 특수문자 normalize
- 괄호 normalize

예:
- FM장비
- FM 장비
- FM-장비

유사 비교 가능하게.

--------------------------------------------------

중요:
원본 상품명은 그대로 보존.

==================================================
6. Import 시 taxonomy 자동 추천
==================================================

상품명 기반으로:
유형 추천 가능하게 한다.

예:
- 배송
- 퀵
→ EX 추천

- 동시통역
- 수행통역
→ IN 추천

- 부스
- 리시버
→ EQ 추천

--------------------------------------------------

중요:
자동 확정 금지.

추천만 제공.

==================================================
7. Import 결과 로그 저장
==================================================

Import 완료 후:
결과 로그 저장.

예:
- import 시간
- 등록 수
- 중복 수
- 검토 필요 수
- 사용자
- 파일명

--------------------------------------------------

향후:
감사 로그/rollback 기반 준비.

==================================================
8. Import UI 방향
==================================================

중요:
ERP형 대량 grid 금지.

방향:
- compact SaaS
- preview 중심
- 검토 workflow
- progressive disclosure

유지.

--------------------------------------------------

즉:
“상품 import 검수 툴”
느낌으로 간다.

==================================================
9. 기존 데이터 보호
==================================================

기존 products 데이터:
자동 삭제 금지.

중복이어도:
자동 merge 금지.

반드시:
운영자가 검토 후 판단 가능하게.

==================================================
10. 커스텀 상품 연결 준비
==================================================

향후:
custom product 요청과 import 구조 연결 가능하도록 유지.

예:
- 자주 요청된 custom product
→ import 후보 전환 가능.

이번 단계에서는:
구조 연결 가능성만 유지.

==================================================
11. 최종 확인
==================================================

확인할 것:

1.
엑셀 업로드 시 preview가 먼저 표시되는지

2.
Dry-run이 실제 INSERT 없이 동작하는지

3.
중복 후보 탐지가 정상 동작하는지

4.
검토 필요 항목이 분리되는지

5.
taxonomy 자동 추천이 정상 동작하는지

6.
기존 products 데이터 손실 없는지

7.
기존 상품관리 workflow 정상 동작하는지

8.
Babel parse / Vite HMR 오류 없는지 확인.
