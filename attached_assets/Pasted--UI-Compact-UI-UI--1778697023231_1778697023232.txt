지시문

[견적 입력 UI Compact 마감 정리 작업]

작업 목적:
현재 견적 입력 UI는 구조적으로 상당히 정리되었으나,
아직 일부 UI 디테일에서:

상세 버튼이 과도하게 길어 보이는 문제
펼침 영역 구분감 부족
row 내부 width 비율 불균형
시각적 밀도 부족

이 남아 있다.

이번 작업은:
“기능 추가”가 아니라
“UI 마감 디테일 정리” 단계이다.

목표:

더 compact하게
더 실무 입력기처럼
더 빠르게 입력 가능하게
입력창처럼 보이는 요소 제거
시각적 안정감 향상

중요:
DB/API/PDF/저장 구조 변경 금지.
오직 UI 디테일만 수정.

───────────────────────

상세 버튼 Compact width 축소
───────────────────────

현재 문제:

상세 버튼:
[ 상세 ▼ ]

가 row 폭을 과도하게 차지하고 있음.

입력창처럼 보이는 원인 중 하나.

변경:

상세 버튼 width auto 또는 고정 compact width 적용.

권장:

width: 72~90px
px-2 ~ px-3
inline-flex
justify-center
gap-1

예시:
[상세 ▼]

주의:

전체 row width 절반 이상 차지 금지
input 스타일 금지
button/pill 느낌 유지

적용 대상:

번역 row
통역 row
장비 row
실비 row

───────────────────────

펼침(showDetail) 영역 배경 구분 추가
───────────────────────

현재 문제:

펼침된 상세영역과 기본 row 영역의 구분감 부족.

변경:

showDetail 영역에 약한 구분감 추가.

권장 중 하나 적용:

background:
#fafafa

또는

border-top:
1px dashed #e5e7eb

또는

rounded + inset 느낌 padding

목표:

“펼쳐진 상세영역”이라는 시각적 인지 강화.

주의:
과한 카드 스타일 금지.
아주 은은하게만 처리.

───────────────────────

통역 row width 재조정
───────────────────────

현재:
언어/방향/방식 dropdown 폭이 약간 큼.

축소 대상:

언어A
언어B
방향
통역방식
단위
수량 input

목표:

한 줄 밀도 향상.

주의:
모바일 width 깨짐 없이 유지.

───────────────────────

번역 row compact 정렬 보정
───────────────────────

현재:
출발어/도착어 input 폭이 조금 큼.

조정:

출발어 width 축소
도착어 width 축소
단위 dropdown width 축소

공급가액 영역 alignment 통일.

───────────────────────

장비 row 최종 정렬 통일
───────────────────────

현재 구조 유지:
[단위] [사용일수] [수량] [단가] [공급가액] [상세]

다만:

input height
spacing
숫자 input width
공급가액 width

를 전체 row 기준으로 통일.

특히:
FM장비 / 송신기 / 수신기 반복 시
줄 정렬이 흐트러지지 않게.

───────────────────────

상세 펼침 애니메이션 최소화
───────────────────────

현재 펼침 시 약간 무거운 느낌 가능.

변경:

duration 짧게
opacity transition 최소화
translate 효과 제거
빠르게 펼쳐지게

목표:
“빠른 입력기 UX”

───────────────────────

버튼 시각 계층 최종 정리
───────────────────────

유형 버튼:
번역 / 통역 / 장비 / 실비

현재보다 더 단순하게.

미선택:

text-gray-400 수준
border transparent
hover 최소

선택:

현재 색 유지 가능
다만 shadow 제거
border 두께 최소화

목표:
“선택된 것만 보이게”
