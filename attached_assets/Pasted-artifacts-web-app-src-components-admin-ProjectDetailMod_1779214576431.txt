artifacts/web-app/src/components/admin/ProjectDetailModal.tsx 파일의
견적 수정/재생성 form 내 상품 선택 dropdown UX를 개선한다.

목표:
현재 상품 선택 UI는 등록된 상품을 단순 리스트로만 보여주고 있어,
상품 수가 많아질 경우 스크롤로 찾기 어려워진다.

앞으로 상품 수가 수백 개 이상 증가할 수 있으므로,
상품 선택 구조를
“스크롤 탐색”
이 아니라
“검색 기반 선택”
방식으로 개선한다.

핵심 목표:
- 상품 검색 속도 향상
- 반복 견적 작성 업무 효율 향상
- 긴 dropdown 스크롤 피로 감소
- SaaS 운영형 workflow 강화

==================================================
현재 문제
==================================================

현재:
- dropdown open
- 전체 상품 리스트 노출
- 스크롤로 직접 탐색

문제:
- 상품 많아질수록 비효율
- 원하는 상품 위치 기억 필요
- 반복 작업 피로 증가
- 작업 속도 저하

==================================================
수정 방향
==================================================

1. 상품 dropdown 내부 상단에 검색 input 추가

placeholder 예시:
- 상품 검색...
- 상품명 / 유형 / 언어쌍 검색

기존 dropdown 구조 안에서 동작하도록 한다.

새 모달/팝업 생성 금지.

==================================================
2. 실시간 필터링 추가
==================================================

입력 시 즉시 필터링.

검색 대상:
- 상품명
- 서비스 유형
- 언어쌍
- 단위
- 단가 문자열
- 기타 표시 텍스트

예:
- "동시"
- "영어"
- "FM"
- "페이지"

입력 시 관련 상품만 노출.

==================================================
3. 검색 결과 리스트 구조 유지
==================================================

현재 상품 리스트 스타일은 유지.

즉:
- 기존 compact dropdown 느낌 유지
- ERP식 table/grid 구조로 변경 금지
- 카드형 상품 selector 금지

==================================================
4. 검색 결과 없음 처리
==================================================

검색 결과가 없으면:

예:
"검색 결과가 없습니다."

문구 표시.

==================================================
5. 기존 상품 선택 로직 유지
==================================================

상품 선택 시 기존 동작 유지:

- 상품명
- 단가
- 단위
- 언어쌍
- interpretationDirection
- quantityUnit
- usagePeriod

자동 입력 로직 유지.

기존 payload / 복원 / 계산 로직 수정 금지.

==================================================
6. UX 기준 유지
==================================================

- dropdown 높이 과도 증가 금지
- 스크롤 구조 유지
- 기존 compact workflow 유지
- vertical spacing 증가 금지
- opacity/transform/blur animation 추가 금지

==================================================
7. 장기 확장 고려
==================================================

이번 작업은 검색 input만 우선 구현.

향후 확장 가능하도록 구조를 정리:
- 최근 사용 상품
- 자주 쓰는 상품
- 고객별 추천 상품
- 언어쌍 기반 추천 상품

등을 나중에 추가 가능하게 유지.

완료 후 확인
==================================================

2. 입력 시 실시간 필터링이 되는지 확인한다.

5. dropdown 높이와 compact workflow가 유지되는지 확인한다.
6. Babel parse / Vite HMR 오류 없는지 확인한다.
