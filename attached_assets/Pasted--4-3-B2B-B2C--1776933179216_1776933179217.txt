[작업명]
견적서 유형 구조 개편 - 4종류를 3종류로 정리

[목표]
현재 견적서 유형이
- B2B 견적서
- B2C 견적서
- 차감 견적서
- 누적 견적서

이렇게 4종류로 구성되어 있는데,
견적서 유형과 청구 방식의 역할이 섞여 있어 구조가 혼란스럽다.

이를 아래 3종류로 정리한다.

최종 견적서 유형:
1. 일반 견적서
2. 선입금 견적서
3. 누적 견적서

즉,
“후불/선결제/차감”은 견적서 유형이 아니라 청구 방식에서 처리하고,
견적서 유형은 문서 구조 기준으로 단순화한다.

--------------------------------------------------
1. 구조 원칙
--------------------------------------------------

[견적서 유형 = 문서 유형]
- 일반 견적서
- 선입금 견적서
- 누적 견적서

[청구 방식 = 돈 흐름]
- 건별 후불
- 선결제(카드/현금)
- 선입금 차감
- 누적 청구

즉,
후불/선결제/차감은 “청구 방식”에서 선택하고,
견적서 유형은 문서 형식만 결정하게 한다.

--------------------------------------------------
2. 견적서 유형 명칭 변경
--------------------------------------------------

기존 → 변경

- B2B 견적서 → 일반 견적서
- B2C 견적서 → 선입금 견적서
- 차감 견적서 → 제거
- 누적 견적서 → 유지

최종 드롭다운 항목은 아래 3개만 남긴다.

- 일반 견적서
- 선입금 견적서
- 누적 견적서

--------------------------------------------------
3. 핵심 규칙
--------------------------------------------------

1) “차감 견적서”는 완전히 제거한다
이유:
차감은 문서 유형이 아니라 청구 방식이기 때문이다.

2) “일반 견적서”는
청구 방식으로
- 건별 후불
- 선결제(카드/현금)
둘 다 가질 수 있어야 한다.

3) “선입금 견적서”는
청구 방식으로
- 선입금 차감
과 연결되는 구조를 기본으로 한다.

4) “누적 견적서”는
청구 방식으로
- 누적 청구
와 연결되는 구조를 기본으로 한다.

--------------------------------------------------
4. UI 수정 대상
--------------------------------------------------

수정 대상 화면:
프로젝트 상세 > 견적/결제 탭 > 견적 생성 영역

수정 내용:
- 견적서 유형 드롭다운 항목 변경
- 안내 문구/설명 문구 변경
- 기존 “차감 견적서” 관련 label 제거

--------------------------------------------------
5. 내부 값(value) 처리 원칙
--------------------------------------------------

중요:
DB 저장값/enum은 가능한 한 최소 변경으로 처리할 것.

권장 방식:
- 기존 내부 enum/value는 유지 가능
- 화면 표시(label)만 우선 변경 가능
- 단, 차감 견적서는 UI에서 선택 불가 처리

예시:
- B2B_GENERAL → 화면에서는 “일반 견적서”
- B2C_PREPAID → 화면에서는 “선입금 견적서”
- BATCH → 화면에서는 “누적 견적서”
- PREPAID_DEDUCT → 신규 생성용에서는 제거 또는 숨김 처리

즉,
기존 데이터 호환성을 해치지 않으면서
신규 생성 기준만 3종류로 정리한다.

--------------------------------------------------
6. 청구 방식 연동 규칙
--------------------------------------------------

견적서 유형 선택에 따라
청구 방식 드롭다운의 기본값 또는 허용값을 아래처럼 정리한다.

[일반 견적서]
허용 청구 방식:
- 건별 후불
- 선결제(카드/현금)

[선입금 견적서]
허용 청구 방식:
- 선입금 차감

[누적 견적서]
허용 청구 방식:
- 누적 청구

권장 UI 동작:
- 견적서 유형 선택 시 청구 방식 자동 기본값 세팅
- 허용되지 않는 청구 방식은 자동 변경 또는 비활성화

예:
일반 견적서 선택 시
→ 기본값: 건별 후불

선입금 견적서 선택 시
→ 자동: 선입금 차감

누적 견적서 선택 시
→ 자동: 누적 청구

--------------------------------------------------
7. 기존 데이터 표시 규칙
--------------------------------------------------

이미 생성된 과거 견적 데이터 중
기존 “차감 견적서” 타입이 있다면
화면 표시 시 아래 원칙으로 처리한다.

권장:
- 신규 생성에서는 선택 불가
- 기존 데이터 상세/리스트/PDF에서는
  “선입금 견적서” 또는 별도 레거시 표시 정책으로 매핑

우선순위:
1) 신규 생성 구조 정리
2) 기존 데이터 깨지지 않게 표시
3) DB 마이그레이션은 이번 작업에서 필수 아님

--------------------------------------------------
8. PDF / 문서 출력 반영
--------------------------------------------------

견적서 PDF에서도 유형 표기를 아래 기준으로 통일한다.

- 일반 견적서
- 선입금 견적서
- 누적 견적서

단,
문서 상단의 가장 큰 제목은 계속
“견적서”
로 유지한다.

즉,
“일반 견적서 / 선입금 견적서 / 누적 견적서”를
문서 제목 자체로 크게 넣지 말고,
배지/보조 라벨/구분 정보 등으로만 표시할 것.

중요:
“후불 견적서”, “선불 견적서” 같은 표현은 문서 제목에 쓰지 않는다.

--------------------------------------------------
9. 설명 문구(툴팁/서브텍스트) 정리
--------------------------------------------------

드롭다운 설명 문구가 있다면 아래처럼 정리:

- 일반 견적서 → 일반 프로젝트에 사용하는 기본 견적
- 선입금 견적서 → 선입금 잔액 기반으로 사용하는 견적
- 누적 견적서 → 월별 또는 기간별 누적 청구용 견적

청구 방식 설명:
- 건별 후불 → 작업 후 건별 청구
- 선결제(카드/현금) → 결제 완료 후 진행
- 선입금 차감 → 선입금 잔액에서 차감
- 누적 청구 → 기간 단위로 모아 청구

--------------------------------------------------
10. 적용 범위
--------------------------------------------------

다음 위치에 모두 반영:

- 견적 생성 드롭다운
- 견적 리스트
- 견적 상세
- PDF 출력
- 유형 label 표시가 있는 모든 화면

기존 명칭:
- B2B 견적서
- B2C 견적서
- 차감 견적서

위 명칭이 화면에 남지 않도록 전체 점검할 것.

--------------------------------------------------
11. 테스트
--------------------------------------------------

반드시 아래 테스트 수행:

1) 견적서 유형 드롭다운에 3개만 노출되는지
- 일반 견적서
- 선입금 견적서
- 누적 견적서

2) “차감 견적서”가 신규 생성에서 사라졌는지

3) 일반 견적서 선택 시
청구 방식이
- 건별 후불
- 선결제(카드/현금)
만 동작하는지

4) 선입금 견적서 선택 시
청구 방식이 선입금 차감으로 연결되는지

5) 누적 견적서 선택 시
청구 방식이 누적 청구로 연결되는지

6) 기존 과거 데이터가 화면에서 깨지지 않는지

7) PDF 출력에서 명칭이 통일되었는지

--------------------------------------------------
12. 완료 기준
--------------------------------------------------

다음 조건을 모두 만족하면 완료:

- 견적서 유형이 3종류로 정리된다
- 차감 견적서는 신규 생성에서 제거된다
- 후불/선결제/차감/누적은 청구 방식에서 처리된다
- 모든 UI와 PDF에서 명칭이 일관되게 반영된다
- 기존 데이터는 깨지지 않는다
