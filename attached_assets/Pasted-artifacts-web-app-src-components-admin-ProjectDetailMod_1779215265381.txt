artifacts/web-app/src/components/admin/ProjectDetailModal.tsx 파일의
견적 수정/재생성 form 내 상품 선택 dropdown UX를 개선한다.

목표:
현재 상품 선택 dropdown은
전체 상품 리스트를 단순 스크롤 방식으로 보여주고 있다.

상품 수가 앞으로 수백 개 이상 증가할 수 있으므로,
상품 선택 구조를:

“전체 리스트 탐색”

이 아니라

“유형 필터 + 검색 기반 selector”

방식으로 개선한다.

핵심 목표:
- 상품 검색 속도 향상
- 반복 견적 작성 workflow 최적화
- 긴 스크롤 탐색 피로 감소
- 실제 운영형 SaaS workflow 강화

==================================================
현재 구조
==================================================

현재:
- dropdown open
- 전체 상품 리스트 노출
- 검색 input 있음
- 하지만 유형 필터 없음

문제:
- 통역 작업 중에도 번역/장비/실비 상품이 모두 같이 보임
- 상품 많아질수록 검색 효율 저하
- 원하는 상품 찾는 속도 느림

==================================================
수정 방향
==================================================

1. 상품 유형 필터 chip 추가
==================================================

상품 검색 input 근처에
compact filter chip/button 추가.

예시 구조:

전체 | 📄 번역 | 🎤 통역 | 🔧 장비 | 💰 실비

텍스트 + 아이콘 함께 사용.

아이콘만 사용하는 구조 금지.

==================================================
2. 기본 상태
==================================================

기본 선택:
- 전체

즉:
dropdown open 시
초기에는 모든 상품 표시.

==================================================
3. 필터 동작 방식
==================================================

[전체]
- 모든 상품 검색/표시

[번역]
- 번역 상품만 표시/검색

[통역]
- 통역 상품만 표시/검색

[장비]
- 장비 상품만 표시/검색

[실비]
- 실비 상품만 표시/검색

==================================================
4. 검색 + 유형 필터 동시 적용
==================================================

예:

통역 선택
+
검색어:
"동시"

↓

동시통역 상품만 표시.

즉:
선택된 유형 범위 안에서 검색 동작.

==================================================
5. 기존 상품 선택 로직 유지
==================================================

상품 선택 시 기존 동작 유지:

- 상품명
- 단가
- 단위
- 언어쌍
- interpretationDirection
- quantityUnit
- usagePeriod

자동 입력 로직 유지.

payload / 계산 / 복원 로직 수정 금지.

==================================================
6. UI 방향
==================================================

중요:
ERP형 상품 selector로 변경하지 않는다.

금지:
- table/grid selector
- 대형 popup
- 새 modal/dialog
- 카드형 selector

현재 compact dropdown workflow 유지.

==================================================
7. UX 목표
==================================================

현재:
- 상품 목록 탐색

변경 목표:
- 상품 빠른 호출

즉:
실무자가:

1.
유형 선택

2.
몇 글자 검색

3.
Enter 또는 클릭 선택

정도로 빠르게 작업 가능하게 한다.

==================================================
8. 향후 확장 가능성 고려
==================================================

향후:
- 최근 사용 상품
- 자주 쓰는 상품
- 고객별 추천 상품
- 언어쌍 기반 추천 상품
- AI 추천 상품

등이 추가될 수 있으므로,
유형 필터 + 검색 구조 기반으로 확장 가능하게 유지.

6. dropdown의 compact workflow가 유지되는지 확인한다.
