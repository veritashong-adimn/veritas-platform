사용자관리와 관리자 통계 기능을 고도화해서, 현재 접속중인 사용자 확인과 접속 통계 조회가 가능하도록 구현해라.

[목표]
관리자가 아래를 확인할 수 있어야 한다.

1. 현재 플랫폼에 접속중인 사용자
2. 사용자별 마지막 로그인/마지막 활동 시간
3. 일별/주별/월별/연도별 접속 통계
4. 사용자별 누적 이용 시간 및 로그인 횟수

---

1. 현재 접속중 사용자 표시

---

사용자관리 목록에 아래 항목을 추가해라.

* 온라인 상태
* 마지막 활동 시간
* 마지막 로그인 시간

표시 예시:

* 온라인
* 오프라인
* 3분 전 활동
* 오늘 09:12 로그인

온라인 판정 기준도 구현해라.
권장 기준:

* 마지막 활동 시간이 최근 N분 이내이면 온라인
* N값은 우선 5분 또는 10분 기준으로 구현

---

2. 활동 로그/세션 로그 저장

---

접속 통계를 위해 로그인 및 활동 로그를 저장하는 구조를 추가해라.

최소 저장 항목:

* user_id
* role_type
* login_at
* logout_at nullable
* last_activity_at
* session_id
* ip_address nullable
* user_agent nullable

가능하면 아래도 고려:

* total_active_seconds
* is_online
* date_key (일별 집계용)

중요:

* 단순 로그인 성공만 저장하지 말고
* 마지막 활동 시간도 갱신되게 해라

---

3. 활동 감지 방식

---

프론트 또는 API 요청 기준으로 활동 감지 로직을 넣어라.

예:

* 페이지 이동
* 주요 API 호출
* 일정 주기 heartbeat
* 화면 활성 상태 확인

권장:

* 로그인 후 heartbeat 또는 API activity 갱신 방식 추가
* 마지막 활동 시간(last_activity_at) 갱신
* 일정 시간 비활성이면 오프라인 처리

---

4. 사용자관리 화면 반영

---

사용자관리 목록에 아래 컬럼 또는 표시 영역을 추가:

* 접속 상태
* 마지막 로그인
* 마지막 활동

필요하면 너무 복잡하지 않게 컬럼 대신 뱃지/툴팁 형식도 가능

예:

* 상태 컬럼: 온라인 / 오프라인
* 마지막 로그인 컬럼
* 마지막 활동 컬럼

---

5. 접속 통계 화면 추가

---

관리자용 접속 통계 화면 또는 대시보드 위젯을 추가해라.

최소 제공 항목:

* 오늘 접속자 수
* 이번 주 접속자 수
* 이번 달 접속자 수
* 올해 접속자 수

추가 제공 항목:

* 일별 로그인 수
* 주별 로그인 수
* 월별 로그인 수
* 연도별 로그인 수
* role_type별 접속 분포
* 사용자별 누적 이용 시간
* 사용자별 로그인 횟수

---

6. 기간 필터

---

통계 화면에서 아래 기간 기준으로 조회 가능하게 해라.

* 오늘
* 이번 주
* 이번 달
* 올해
* 사용자 지정 기간(가능하면)

---

7. 이용 시간 계산 방식

---

이용 시간은 단순 로그인~로그아웃만 보지 말고,
가능하면 마지막 활동 기준으로 세션 시간을 계산해라.

최소 기준:

* login_at ~ last_activity_at 또는 logout_at 기준 계산

보고 시 아래를 구분해서 설명:

* 로그인 기준 체류시간
* 활동 기준 추정 이용시간

---

8. 권한

---

이 기능은 관리자(admin) 중심으로 제공해라.
직원(staff)은 필요시 읽기 전용 일부만 허용 가능하지만,
우선은 admin 전용으로 구현해라.

---

9. API 추가

---

필요한 API를 설계/추가해라.

예시:

* 현재 온라인 사용자 목록 조회
* 사용자별 최근 접속 정보 조회
* 접속 통계 조회
* heartbeat/activity update API
