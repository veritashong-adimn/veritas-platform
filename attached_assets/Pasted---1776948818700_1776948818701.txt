[작업명]
담당자 상세 모달 - 수정 기능 추가

[목표]
현재 담당자 등록과 상세 조회는 가능하지만,
담당자 상세 모달에서 수정 버튼이 보이지 않아 기존 담당자 정보를 변경할 수 없다.

이를 개선하여
담당자 상세 모달에서
- 수정 버튼 노출
- 편집 모드 전환
- 저장 / 취소
가 가능하도록 한다.

--------------------------------------------------
1. 문제
--------------------------------------------------

현재 담당자 상세 모달에는
- 이름
- 거래처
- 이메일
- 전화
- 부서
- 직책
등 정보가 보이지만,

수정 버튼이 없어
등록 후 정보 변경이 불가능하다.

실무에서는 담당자 정보 수정이 자주 발생하므로
상세 모달에서 바로 수정할 수 있어야 한다.

--------------------------------------------------
2. 수정 대상
--------------------------------------------------

화면:
고객/거래처 > 담당자 관리 > 담당자 상세 모달

예상 파일:
- ContactDetailModal.tsx
- ContactsPage.tsx
- ContactModal.tsx
또는 담당자 상세 관련 컴포넌트

실제 파일명을 검색해서 정확한 컴포넌트에서 수정할 것.

검색 키워드:
- 담당자 상세
- 연락처 상세
- contact detail
- contact modal

--------------------------------------------------
3. UI 수정
--------------------------------------------------

담당자 상세 모달 우측 상단 또는 제목 영역에
아래 버튼 추가:

[수정]

닫기(X) 버튼과 함께 자연스럽게 배치

예시:
담당자 상세                         [수정] [X]

--------------------------------------------------
4. 수정 모드 동작
--------------------------------------------------

기본 상태:
보기 모드 (read only)

[수정] 클릭 시:
편집 모드로 전환

편집 가능 필드:
- 이름
- 부서
- 직책
- 이메일
- 전화
- 휴대폰
- 메모(있다면)

읽기 전용 유지 권장:
- 거래처 (company)
  → 담당자 소속 거래처 변경은 별도 정책이 필요하므로 이번 작업에서는 수정 대상 제외 권장

즉,
이번 작업은 “담당자 기본 정보 수정”에 집중

--------------------------------------------------
5. 편집 UI
--------------------------------------------------

현재 텍스트 표시 방식:
이름: 최지윤
이메일: xxx@xxx.com

편집 모드에서는 입력 필드로 전환:

이름: [input]
이메일: [input]
전화: [input]
부서: [input]
직책: [input]

모달 하단 또는 우측 하단에 버튼 표시:

[취소] [저장]

--------------------------------------------------
6. 저장 동작
--------------------------------------------------

저장 시 PATCH API 호출

예상 API:
PATCH /api/admin/contacts/:id
또는 현재 담당자 수정 API 경로에 맞춰 연결

payload 예시:
{
  "name": "최지윤",
  "department": "마케팅팀",
  "position": "과장",
  "email": "new@email.com",
  "phone": "02-1234-5678",
  "mobile": "010-1234-5678"
}

성공 시:
- 저장 완료 toast 표시
- 상세 모달 즉시 갱신
- 목록에도 반영

실패 시:
- 오류 메시지 표시
- 기존 입력 유지

--------------------------------------------------
7. 검증
--------------------------------------------------

아래 검증 적용:

- 이름 필수
- 이메일 형식 검증
- 전화/휴대폰 포맷은 너무 엄격하지 않게 허용
- 빈 문자열은 null 또는 공란으로 정리

--------------------------------------------------
8. 취소 동작
--------------------------------------------------

[취소] 클릭 시:
- 수정 내용 폐기
- 보기 모드로 복귀

모달 전체 닫기 없이 보기 모드로 돌아가게 처리하는 방식 권장

--------------------------------------------------
9. 로그/Audit
--------------------------------------------------

가능하면 담당자 수정 시 로그 남길 것.

예:
- contact_updated
- old/new 값 비교 가능하면 함께 기록

최소한:
누가, 언제, 어떤 담당자를 수정했는지 남길 것

--------------------------------------------------
10. 영향 범위
--------------------------------------------------

이번 작업에서는 아래는 제외:

- 담당자 삭제 기능
- 거래처 변경 기능
- 고객 로그인 계정 초대/연결
- 기본 담당자 지정 변경

우선은
“상세 모달에서 담당자 기본 정보 수정 가능”
까지만 완성

--------------------------------------------------
11. 테스트
--------------------------------------------------

반드시 아래 테스트 수행:

1) 담당자 상세 모달에 [수정] 버튼이 보이는지
2) 수정 클릭 시 입력 필드로 전환되는지
3) 이름/이메일/전화 수정 후 저장되는지
4) 저장 후 목록과 상세 화면이 갱신되는지
5) 취소 시 변경사항이 버려지는지
6) 이메일 형식 오류 시 저장 차단되는지

--------------------------------------------------
12. 완료 기준
--------------------------------------------------

다음이 모두 만족되면 완료:

- 담당자 상세 모달에 수정 버튼이 보인다
- 수정 모드 전환이 된다
- 저장/취소가 정상 동작한다
- 수정 내용이 API/DB에 반영된다
- 목록/상세 화면 모두 갱신된다
