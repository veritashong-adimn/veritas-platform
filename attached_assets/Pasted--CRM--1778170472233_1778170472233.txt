지시문

견적 입력 화면의 상단 구조를 실무형 CRM 기준으로 단순화한다.

현재 문제:

* 견적 입력 화면에 너무 많은 정보가 동시에 노출됨
* 회계/세무/청구 정보까지 한 화면에 들어와 복잡함
* “견적유효기간”, “발행예정일” 등 실무 흐름과 맞지 않는 항목이 있음
* 사용자는 실제로:
  상품/수량/금액
  중심으로 빠르게 견적을 만들고 싶어함

목표:
견적 입력 UI를 단순화하고,
회계/세무 정보는 자동화 또는 숨김 처리하여
실무형 빠른 견적 생성 구조로 변경.

━━━━━━━━━━━━━━━━━━

1. 견적유효기간 UI 제거
   ━━━━━━━━━━━━━━━━━━

현재:

* 견적유효기간 입력 UI 존재
* “발행일 +30일” 문구 표시

변경:
견적유효기간 입력 UI 제거.

대신:
“견적일”만 표시.

━━━━━━━━━━━━━━━━━━
2. 견적일 구조 변경
━━━━━━━━━━━━━━━━━━

UI:

* 견적일 필드만 표시

시스템 내부:
자동 계산.

기준:
견적일 + 30일

예:
2026-05-07
→ 내부 validUntil:
2026-06-06 자동 계산

사용자에게는 표시하지 않음.

━━━━━━━━━━━━━━━━━━
3. “발행일 +30일” 문구 제거
━━━━━━━━━━━━━━━━━━

현재 표시되는:
“발행일 +30일”
문구 제거.

이유:
UI 복잡도 증가 요소이며
사용자 입력 필요 없는 내부 로직이기 때문.

━━━━━━━━━━━━━━━━━━
4. 발행예정일 제거
━━━━━━━━━━━━━━━━━━

견적 입력 화면에서:
“발행예정일” 완전 제거.

이유:
세금계산/청구 단계 정보이므로
견적 입력 단계와 분리 필요.

━━━━━━━━━━━━━━━━━━
5. 입금예정일 이동
━━━━━━━━━━━━━━━━━━

견적 입력 화면에서 제거.

이동 위치:

* 결제관리 탭
  또는
* 결제 등록 영역

이유:
프로젝트 재무관리 정보이기 때문.

━━━━━━━━━━━━━━━━━━
6. 문서구분 / 발행유형 / 청구방식 처리 변경
━━━━━━━━━━━━━━━━━━

현재:
항상 노출.

변경:
기본 숨김 또는 접기 패널 처리.

예:
[회계/세무 옵션]

안으로 이동.

━━━━━━━━━━━━━━━━━━
7. 거래처 기반 자동 기본값 적용
━━━━━━━━━━━━━━━━━━

국내 거래처:

* 세금계산서
* 일반
* 건별후불

자동 기본 적용.

해외 거래처:

* 영세율
* invoice

자동 적용 가능하도록 구조 준비.

━━━━━━━━━━━━━━━━━━
8. 견적 입력 화면 핵심 UX 유지
━━━━━━━━━━━━━━━━━━

견적 입력 메인 화면은:

* 상품 선택
* 수량
* 단가
* 금액

중심으로 단순 유지.

실무자가 빠르게 견적 생성 가능해야 함.

━━━━━━━━━━━━━━━━━━
9. 기존 데이터 영향 금지
━━━━━━━━━━━━━━━━━━

기존:

* 견적 데이터
* PDF
* 프로젝트 상세

깨지지 않게 유지.

내부 validUntil 계산 구조만 자동화.

━━━━━━━━━━━━━━━━━━
10. PDF 출력 유지
━━━━━━━━━━━━━━━━━━

견적서 PDF에는:

“본 견적서는 견적일로부터 30일간 유효합니다.”

자동 출력 유지.

사용자 입력 UI만 제거.

━━━━━━━━━━━━━━━━━━
11. E2E 최소 테스트 원칙 적용
━━━━━━━━━━━━━━━━━━

과도한 전체 E2E 반복 금지.

원칙:

* 관련 파일만 최소 탐색
* data-testid / aria-label 먼저 적용
* 핵심 시나리오만 테스트
* 동일 원인 2회 실패 시 즉시 중단 후 보고
* 전체 로그인 플로우 반복 금지

━━━━━━━━━━━━━━━━━━
12. 반드시 확인할 E2E
━━━━━━━━━━━━━━━━━━

반드시 검증:

* 견적유효기간 UI 제거 확인
* 견적일 정상 표시
* 내부 +30일 자동 계산 확인
* 발행예정일 제거 확인
* 입금예정일 이동 확인
* PDF 정상 출력 확인
* 기존 견적 데이터 정상 표시 확인
* 빌드 오류 없는지 확인

E2E 결과까지 보고.
