거래처 등록/수정 구조를 B2B 실무에 맞게 개편해라.

[문제]
현재 거래처 등록 화면은 회사(company)만 등록할 수 있고,
브랜드사/부서(divisions)와 담당자(contacts)를 등록 시점에 함께 연결할 수 없다.

하지만 실제 운영에서는 아래 구조가 필요하다.

예:

* 회사: 리치몬트코리아
* 브랜드/부서: 반클리프아펠, 몽블랑, 까르띠에
* 담당자: 각 브랜드/부서별 담당자

즉, 거래처 등록 시
회사 선택/입력 → 브랜드사 선택 또는 등록 → 해당 브랜드 담당자 등록
까지 한 흐름으로 처리되어야 한다.

---

1. 거래처 등록 화면 개편

---

거래처 등록 시 아래 구조를 지원해라.

[1단계] 회사 정보

* 거래처명(company)
* 사업자등록번호
* 대표자명
* 대표전화
* 이메일
* 주소
* 업태/종목
* 거래처 유형
* 외주유형(외주업체일 경우)

[2단계] 브랜드사/부서 정보

* 브랜드/부서 선택 또는 신규 등록 가능
* 예:

  * 반클리프아펠
  * 몽블랑
  * 까르띠에

중요:

* 회사 등록 직후 바로 브랜드/부서를 추가할 수 있어야 한다
* “브랜드사 없음”도 가능해야 한다

[3단계] 담당자 정보

* 담당자 이름
* 소속 브랜드/부서 선택
* 직책
* 휴대폰
* 일반전화
* 이메일
* 메모
* 기본 담당자 여부
* 견적 담당 여부
* 청구 담당 여부

즉, 등록 시점에
회사 아래 브랜드/부서를 만들고,
그 브랜드/부서 소속 담당자를 바로 추가할 수 있게 해라.

---

2. 수정 화면도 동일 구조로 정리

---

거래처 상세/수정 화면에서도 아래가 가능해야 한다.

* 회사 정보 수정
* 브랜드/부서 추가/수정/삭제
* 브랜드별 담당자 추가/수정/삭제

즉, 현재 수정 화면의 하단 담당자 등록 영역을
브랜드/부서와 연결된 구조로 바꿔라.

---

3. 담당자와 브랜드 연결

---

contacts 테이블의 divisionId 연결을 실제 UI에 반영해라.

담당자 등록 시 반드시:

* 어떤 브랜드/부서 소속인지 선택 가능해야 한다

예:
홍길동 / 반클리프아펠 담당
김영희 / 몽블랑 담당

---

4. 등록 화면 UX

---

권장 흐름:

A안:

* 회사 정보 입력
* 저장 전, 같은 화면 아래에서 브랜드/부서 추가
* 브랜드 추가 후 담당자 추가

B안:

* 회사 저장 후 즉시 “브랜드/담당자 계속 등록” 섹션 노출

중요:

* 사용자가 새 창을 다시 열지 않고 한 번에 등록 가능해야 한다

---

5. 거래처 상세 화면 표시 개선

---

상세 화면에서 아래 구조가 명확히 보여야 한다.

[회사]
리치몬트코리아

[브랜드/부서]

* 반클리프아펠
* 몽블랑
* 까르띠에

[담당자]

* 홍길동 / 반클리프아펠 / 견적 담당
* 김영희 / 몽블랑 / 청구 담당

즉, 담당자 목록에 division 이름이 반드시 표시되어야 한다.

---

6. 견적서/프로젝트 표기 규칙

---

견적서 및 프로젝트 화면에서
브랜드/부서가 있는 경우 아래처럼 표기해라.

표기 규칙:

* 회사만 있으면: 리치몬트코리아
* 브랜드/부서가 있으면: 리치몬트코리아(반클리프아펠)

프로젝트 목록/상세/견적 PDF/견적 미리보기 모두 동일 규칙 적용

---

7. 프로젝트 등록 화면도 연결

---

프로젝트 등록 시
아래 순서로 선택 가능해야 한다.

* 요청 회사 선택
* 요청 브랜드/부서 선택
* 요청 담당자 선택

즉,
company → division → contact
순으로 종속 선택되게 해라.

---

8. API/DB 점검

---

아래 구조 실제 연결 확인:

* companies
* divisions
* contacts.divisionId

필요 시 수정:

* 거래처 생성 API
* 브랜드/부서 생성 API
* 담당자 생성 API
* 거래처 상세 조회 API
* 거래처 목록 조회 API
* 프로젝트/견적 조회 API

---

9. 기존 데이터 호환

---

기존에 division 없이 등록된 거래처도 깨지지 않게 유지
기존 담당자는 divisionId가 null이어도 조회 가능
단, 신규 등록부터는 브랜드 연결을 쉽게 할 수 있게 UI 개선
