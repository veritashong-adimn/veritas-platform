[목표]
플랫폼 전반에 적용되는 역할 기반 권한 관리 시스템(Role-Based Access Control, RBAC)을 구축합니다.

기존 기능(프로젝트, 거래처, 담당자, 정산 등)에 대해
화면 접근 권한과 기능 실행 권한을 모두 제어할 수 있도록 합니다.

==================================================
1. 핵심 구조
==================================================

권한 구조는 아래 3단계로 구성합니다.

사용자(User) → 역할(Role) → 권한(Permission)

- 사용자는 하나의 역할(Role)을 가짐
- 역할은 여러 권한(Permission)을 가짐

==================================================
2. DB 스키마 설계
==================================================

[roles 테이블]
- id
- name (예: admin, pm, finance)
- description
- created_at

[permissions 테이블]
- id
- key (예: project.create, settlement.pay)
- name
- category (menu / action)
- created_at

[role_permissions 테이블]
- id
- role_id
- permission_id

[users 테이블 수정]
- role_id 컬럼 추가

==================================================
3. 권한 종류 정의
==================================================

권한은 2가지로 구분합니다.

----------------------------------
1) 화면 권한 (Menu Access)
----------------------------------

category = menu

예:
- menu.dashboard
- menu.project
- menu.task
- menu.payment
- menu.settlement
- menu.prepaid
- menu.company
- menu.contact
- menu.customer
- menu.translator
- menu.product
- menu.user
- menu.permission

----------------------------------
2) 기능 권한 (Action Permission)
----------------------------------

category = action

예:
- project.create
- project.update
- project.delete
- project.view

- quote.create
- quote.approve
- quote.update

- settlement.view
- settlement.pay

- company.create
- company.update

- contact.create
- contact.update

==================================================
4. 백엔드 권한 체크 (중요)
==================================================

모든 주요 API에 권한 체크를 추가합니다.

예:
- 프로젝트 생성 → project.create 필요
- 정산 지급 → settlement.pay 필요

권한이 없으면:
→ 403 Forbidden 반환

==================================================
5. 프론트엔드 권한 적용
==================================================

1. 화면 권한
- 권한 없는 메뉴는 사이드바에서 숨김

2. 기능 권한
- 권한 없는 버튼 숨김 또는 비활성

예:
- 정산 지급 버튼 → settlement.pay 없으면 숨김

==================================================
7. 기본 역할 정의 (초기 세팅)
==================================================

admin
- 모든 권한 허용

pm
- 프로젝트 / 견적 / 담당자 관리
- 정산 지급 제외

finance
- 정산 / 결제
- 프로젝트 수정 제한

viewer
- 조회만 가능

==================================================
8. 보안 규칙
==================================================

- 프론트 숨김만으로 권한 처리하지 않음
- 반드시 백엔드에서 권한 검증 수행
- 관리자만 권한 관리 접근 가능

==================================================
9. 목표
==================================================

실무형 권한 시스템을 구축합니다.
