| 필드 | 필수/선택 | 비고 |
|------|----------|------|
| 결제 금액 | ✅ 필수 | approved 상태에서만 등록 가능 |

#### 정산 (Settlement)

| 필드 | 필수/선택 | 비고 |
|------|----------|------|
| 총액 | ✅ 자동 | 결제 금액과 동일 |
| 번역사 지급액 | ✅ 자동 | 총액 × 80% (플랫폼 수수료 20%) |
| 플랫폼 수수료 | ✅ 자동 | 총액 × 20% |
| 정산 상태 | ✅ | pending → paid |

⚠️ **주의사항**
- 정산은 status→completed + 결제+번역사 존재 시 자동 생성됩니다.
- 수수료율(20%) 변경이 필요한 경우 개발 요청 필요

🔍 **검수 포인트**
- [ ] 견적 금액이 실제 계약 금액과 일치하는지
- [ ] 결제 등록 후 정산이 자동 생성되는지

## 8. 상태 전이 규칙 (Status Transition)

```
접수됨(created) → 견적됨(quoted) → 견적승인됨(approved) → 결제완료(paid) → 매칭됨(matched) → 작업중(in_progress) → 완료(completed)
                                                                                                      ↓ (모든 상태에서 가능)
                                                                                               취소됨(cancelled)
```

| 전이 | 조건 |
|------|------|
| quoted → approved | 수동 변경 |
| approved → paid | 결제 등록 시 자동 OR 수동 변경 (결제 기록 필요) |
| paid → matched | 번역사 배정 후 수동 (번역사 배정 필요) |
| matched → in_progress | 수동 변경 (번역사 배정 필요) |
| in_progress → completed | 수동 변경 → 정산 자동 생성 |

## 9. 운영 체크리스트

### 일일 점검
- [ ] 신규 프로젝트 접수 확인
- [ ] 결제 대기 프로젝트 확인
- [ ] 번역사 배정 대기 프로젝트 확인 (paid 상태)
- [ ] 작업 완료 후 정산 생성 확인

### 신규 프로젝트 처리 흐름
1. 접수됨 → 견적 생성 → 견적됨
2. 고객 승인 확인 → 견적승인됨
3. 결제 등록 → 결제완료
4. 번역사 배정 → 매칭됨
5. 작업 시작 → 작업중
6. 작업 완료 확인 → 완료 → 정산 자동 생성
